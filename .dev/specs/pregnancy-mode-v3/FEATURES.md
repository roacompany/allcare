# 임신모드 v3 — 기능 명세 (전체)

작성: 2026-06-11 · 풀 추적기 스코프 전부 확정(PO). **순수 기능/UX** (데이터모델·코드 제외, 별도 단계). 영역별 병렬 상세화 종합. 광고·커머스 제외(BabyCare 광고0).

## A. 추적 (임신 본격 추적기 — 홈 케어 프로필 카드 + 5탭 분배)

### ① 임신 주차(NN주N일) + D-day

- **무엇을**: 마지막 생리일(LMP) 또는 출산예정일(EDD) 기준으로 '현재 임신 며칠째인지'를 한국식 NN주N일 표기와 출산예정일까지 D-day로 항상 보여주는 임신 추적의 중심축. 홈 케어 프로필 카드 스택의 '임신 카드' 1장이 핵심 표면: 태아 일러스트 + 'D-NN' + '임신 NN주 N일' + 주차별 크기 비유(예: 임신 16주 → '아보카도 정도') + 빠른 액션 버튼. 매일 자정에 자동으로 1일씩 진행한다.
- **어떻게 보이나**: 홈 임신 카드 + 주차 상세 + 캘린더 마커 + 위젯
- **입력**: 온보딩 LMP 또는 EDD 1회 입력, 이후 자동
- **한국 맥락**: 한국 산모수첩·병원 모두 'NN주N일'(예: 24주 3일)과 'D-day'를 표준으로 쓰므로 양쪽 동시 표기 필수. 주차 크기 비유는 한국에서 친숙한 과일/채소·사물 위주(참깨·블루베리·자두·아보카도·바나나·수박 등). 280일 규칙(Naegele)과 '주차=막달까지' 한국 통념을 그대로 따른다. 삼분기 명칭은 '초기·중기·후기'로 표기.
- **엣지/주의**: LMP와 병원 EDD가 다를 때 병원 값 우선 + '예정일이 조정되었어요' 안내. 생리 주기가 불규칙하면 LMP 추정 부정확 → '초음파 검진 후 예정일을 확정하세요' 힌트. 예정일 초과(40주+, D-day가 마이너스로) 시 'D+N 예정일을 지났어요, 막달 진료 안내' 표시로 전환(불안 유발 표현 금지). 42주 초과 등 비정상 장기는 의료 안내 배너. 출생 전환/유산·사산 등 종료 시 카드는 즉시 추적을 멈추고 별도 플로우로(데이터 삭제 절대 금지).

### ② 태동 카운터

- **무엇을**: 아기가 발길질·움직임을 보일 때 탭으로 횟수를 세고, '10번 채우는 데 걸린 시간'을 측정하는 세션형 카운터. 후기 임신의 핵심 안심 도구. 한 세션 = '10회 도달까지' 또는 '정해진 시간 동안'. 결과(시작시각·10회 도달 소요시간·총 횟수)를 기록으로 남겨 날짜별 추이를 본다.
- **어떻게 보이나**: 원형 카운트 + 타이머 + 10회 요약 + 추이 그래프
- **입력**: 원형 탭 카운트 + 세션 시작/종료, 자세·식후 메모(선택)
- **한국 맥락**: 한국 산부인과 표준 안내인 '2시간에 10회'를 기준선으로 사용. 임신 30주 이후 시간당 평균 3회 이하(하루 20회 미만)가 2일 이상 지속되면 병원 방문 권고를 그대로 반영한 부드러운 안내. 보유 KickSession 자산 활용. 태담(아기에게 말 걸기) 문화와 연결해 '태동 느낀 순간 기록'을 정서적 경험으로 제시.
- **엣지/주의**: 보통 태동은 18~25주경 시작 → 너무 이른 주차에선 카운터 강조 대신 '아직 못 느껴도 정상' 안내. 잠든 아기로 한참 움직임 없음 → '간식/물 먹고 옆으로 누워 다시 세보세요' 같은 자극 팁(진단 아님). 장시간(2시간+) 세션의 타이머/햅틱 안정성 보장. 평소보다 현저히 적거나 멈춘 느낌 → '움직임이 줄었다고 느끼면 즉시 병원에 연락하세요' 비응급 표현 배너(앱은 진단/응급 판단을 하지 않음). 0회로 종료된 세션도 기록 보존.

### ③ 체중 추적 + 증가 그래프(임신 전 BMI 권장범위)

- **무엇을**: 임신 전 체중을 기준점으로 현재 체중과 '누적 증가량'을 추적하고, 임신 전 BMI에 따른 권장 총 증가 범위 밴드 위에 내 추이를 겹쳐 보여주는 그래프. '잘 늘고 있는지'를 스스로 가늠하게 한다(진단 아님).
- **어떻게 보이나**: 증가량 요약 + 권장밴드 라인차트 + 권장범위 카드
- **입력**: 임신 전 키·체중 1회 + 측정값 추가, HealthKit opt-in
- **한국 맥락**: 대한산부인과학회 기준의 임신 전 BMI별 권장 총 증가량을 그대로 사용: 저체중(BMI<18.5) 12.5~18kg / 정상(18.5~23) 11.5~15kg / 과체중(23~25) 7~11.5kg / 비만(≥25) 7kg 미만. 한국 비만 BMI 기준(≥25)이 서구(≥30)와 달라 한국 기준 적용이 중요. kg·cm 단위 고정. 산모수첩에 체중을 적는 한국 관행과 연결해 '진료 때 보여주기' 공유/리포트 지원.
- **엣지/주의**: 초기 입덧으로 체중 감소·정체 → '초기엔 늘지 않거나 줄 수 있어요' 정상 안내, 밴드 아래여도 경고 톤 금지. 쌍둥이는 권장범위가 달라 단태아 밴드를 그대로 적용하지 않도록 '다태아는 의료진과 상의' 주석. 임신 전 체중 미입력 시 밴드 없이 추이만 표시. 급격한 단기 증가(부종/임신중독증 가능)는 의학적 판단 텍스트 없이 '진료 시 상의' 수준 안내만. 백분위/의학적 판단 문구 금지 규칙 준수.

### ④ 증상·컨디션·기분 로깅

- **무엇을**: 입덧·요통·부종·피로·속쓰림 등 임신 증상과 그날의 컨디션·기분을 가볍게 태그로 남기는 일기형 로깅. 주차별로 흔한 증상을 추천 칩으로 제시해 한 손 입력. 추이를 모아 '진료 때 의사에게 보여줄 요약'으로 만든다.
- **어떻게 보이나**: 오늘 컨디션 입력 + 증상 빈도/기분 추세 + 진료용 요약
- **입력**: 기분 이모지 + 증상 칩 + 강도 + 메모(선택)
- **한국 맥락**: 한국 임신부가 흔히 검색·기록하는 증상 용어(입덧, 부종, 변비, 치골통/골반통, 분비물, 다리 쥐, 명치 답답함 등)를 칩으로 준비. 보유 PregnancySymptom 자산 확장. 출혈·심한 복통·물 같은 분비물(양수)·심한 두통+시야흐림(임신중독 신호) 등 응급 가능 증상은 입력 시 '즉시 병원/산모수첩의 응급 연락처로 연락' 비진단 안내를 함께 노출.
- **엣지/주의**: 증상 기록은 진단/원인 추정 절대 금지 — 빈도/추이만 시각화. AI 인사이트/Analytics에 임신·증상 데이터 포함 금지(민감 건강정보, safety 규칙). 응급 키워드 증상 선택 시에도 앱이 위험도를 판정하지 않고 '의료진 연락' 고정 문구만. 같은 날 여러 번 기록 허용(누적). 빈 기록(증상 없음)도 '오늘은 편안함'으로 의미 있게 저장.

### ⑤ 진통 간격 타이머

- **무엇을**: 막달에 진통이 시작됐을 때 '수축 시작~끝'을 버튼으로 찍어 진통 간격(주기)과 지속 시간을 자동 계산하는 타이머. '지금이 병원 갈 때인지' 판단을 돕는 5-1-1 기준 안내를 함께 제공(진단 아님).
- **어떻게 보이나**: 수축 토글 + 실시간 간격/지속 표 + 5-1-1 진행 표시
- **입력**: 수축 시작/끝 토글 2탭, 초산/경산 설정, 취소·수정
- **한국 맥락**: 한국 병원 표준 안내 반영: 초산은 수축이 규칙적으로 '5분 간격'이 되면 병원행, 경산은 10분 간격이어도 강하거나 이전 출산과 유사하면 병원행 — '초산/경산' 설정에 따라 기준 안내를 다르게. 5-1-1(5분 간격·1분 지속·1시간 지속) 일반 기준을 제시하되 '병원·조산원마다 다를 수 있으니 안내받은 기준을 따르세요'를 명시. 양수 터짐·출혈 시 간격과 무관하게 즉시 병원 안내.
- **엣지/주의**: 가진통(브랙스턴힉스)과 진진통 구분은 앱이 판정하지 않음 — '불규칙하면 가진통일 수 있어요, 규칙적으로 짧아지면 병원' 정도의 비진단 안내만. 28주 미만에서 규칙적 수축은 조기진통 가능 → '예정일 전 규칙적 진통은 즉시 병원 연락' 우선 안내. 단발 수축/오입력에 평균이 왜곡되지 않도록 최근 윈도우 기준. 세션이 길어져도(수 시간) 안정 동작. 병원행 판단 책임은 사용자/의료진에 있음을 고지.

### ⑥ 혈압·혈당(임당) 기록

- **무엇을**: 가정/병원에서 잰 혈압(수축기/이완기)과 혈당(공복·식후)을 날짜·시간대별로 기록하고 추이로 보는 도구. 임신중독증 관리(혈압)와 임신성 당뇨 관리(혈당) 두 용도. 임당 식이관리 중인 산모가 매끼 혈당을 찍는 일상 동선을 지원.
- **어떻게 보이나**: 혈압·혈당 각 섹션, 목표 참고선 추이 + 진료용 요약
- **입력**: 혈압 2값 + 맥락 / 혈당 값 + 끼니 맥락 칩, HealthKit opt-in
- **한국 맥락**: 한국 임당 관리 목표치를 참고선으로(진단 아님): 공복 95mg/dL 미만, 식후 1시간 140mg/dL 미만, 식후 2시간 120mg/dL 미만. 혈당 단위는 mg/dL(국내 표준), 혈압 mmHg. 24~28주 임당검사(50g 선별→140↑ 시 100g 확진) 일정과 연결해 '검사 예정' 안내. 혈압은 임신중독증(전자간증) 맥락에서 흔히 가정 측정하는 한국 관행 반영.
- **엣지/주의**: 목표치는 '참고선'일 뿐 의학적 판단/경고 문구 금지 — 범위 밖이어도 색 표시+'진료 시 상의'까지만. 임당 진단·치료 결정은 의료진 몫임을 고지. 단위 혼동 방지(mmol/L 미지원 명시 또는 변환 안내). 비정상 고혈압(예: 160/110 이상) 같은 응급 가능 수치도 앱이 위험판정 대신 '즉시 병원 연락' 고정 안내. 혈압/혈당 데이터도 Analytics·AI 인사이트 포함 금지(민감정보). 측정 맥락 누락 시 추이 해석이 흐려지므로 맥락 선택을 권장하되 강제하지 않음.

### ⑦ 약 복용 / 수분 / 수면 (선택 추적)

- **무엇을**: 임산부 영양제·처방약 복용, 하루 수분 섭취, 수면을 가볍게 체크하는 선택형 보조 추적. 본격 의료수치(②~⑥)와 달리 부담 없는 습관 체크 성격. 사용자가 켠 항목만 홈/건강에 노출되는 모듈식.
- **어떻게 보이나**: 모듈식 미니 카드(약 체크·수분 게이지·수면) + 홈 단축
- **입력**: 항목 on/off + 약 1탭 완료 / 물 +1컵 / 수면 시각·HealthKit
- **한국 맥락**: 한국 임산부가 기본으로 챙기는 엽산(초기)·철분/칼슘(중후기) 영양제를 약 항목 기본 추천으로 제시(보유 pregnancy-weeks.json 팁의 '산전 비타민·엽산' 연계). 약물 안전성 판단은 앱이 하지 않고 '복용 전 의료진/약사 상담' 고지(임신부 금기약 다수). 카페인·금기 음식 경고는 ④/주차 팁에서 다루고 여기선 '복용 여부 기록'에 한정.
- **엣지/주의**: 약 안전성/상호작용 판정 절대 금지 — 단순 복용 체크만. 복용 누락에 죄책감 유발 톤 금지(부드러운 리마인더). 수분/수면 목표 미달도 경고 아닌 격려. 세 항목 모두 off가 기본값이라 '본격 추적기'에서도 강요하지 않음(부담 최소화). HealthKit 미연동/거부 시 수동 입력 폴백. 선택 추적 데이터도 Analytics·AI 인사이트 제외(민감정보 보수적 처리).

## 영역 B. 주차별 콘텐츠 — 태아발달/엄마변화/아빠팁 3블록 · 데일리 팁 · 아기 크기 비교 · 발달 일러스트 · 전문가 감수 아티클 라이브러리

### 주차별 3블록 콘텐츠 (태아 발달 / 엄마 변화 / 아빠 팁)

- **무엇을**: 현재 임신 주차에 맞춰 매주 새로 펼쳐지는 콘텐츠 본문의 핵심. 세 개의 독립 블록으로 구성한다. ①태아 발달: pregnancy-weeks.json의 milestone 텍스트를 1블록으로 확장 — '이번 주 아기에게 일어나는 일'을 2~3문장으로(예: 16주 '움직임이 활발해지고 청각이 발달합니다' → 청각 발달과 첫 태동 가능성을 풀어쓴 단락). 신체 길이(머리~엉덩이 CRL 또는 키)·예상 체중을 cm/g로 함께 표시. ②엄마 변화: 이 주차에 흔한 모체 변화·증상을 안내(예: 6주 입덧, 16주 입덧 완화·태동 인지, 28주 허리 통증·부종). '정상 범위'와 '의료진 상담이 필요한 신호'를 톤으로 구분하되 진단은 하지 않는다. ③아빠(파트너) 팁: 이 주차에 배우자가 할 수 있는 구체 행동 1~2개(예: 16주 '태담을 함께 시작해보세요', 32주 '출산 가방을 같이 챙기고 병원 동선을 확인하세요'). 부부공유 자산과 연결돼 두 사람이 같은 콘텐츠를 본다. 세 블록은 카드/섹션으로 시각 구분되고, 주차가 바뀌면 콘텐츠가 자동 교체된다.
- **어떻게 보이나**: 건강 탭의 '임신 주차 콘텐츠' 진입 화면을 메인 무대로 한다(임신 전용 탭 없음). 홈의 임신 카드 '빠른 액션'에서 '이번 주 보기'로 바로 진입. 화면 상단에 NN주N일 + D-day + 태아 일러스트(아래 일러스트 기능과 연동), 그 아래로 3블록을 수직 스택. 좌우 스와이프 또는 상단 주차 페이저로 이전·다음 주를 미리보기/되돌아보기 가능(읽은 주차는 다시 볼 수 있음). 각 블록 하단에 '관련 아티클 더 보기'로 라이브러리 연결. 의학 정보 블록 하단에는 면책 문구를 항상 노출(disclaimerKey 기반 — 일반/태동/진통 맥락별).
- **입력**: 직접 입력 없음. 출산예정일(EDD) 또는 마지막 월경일(LMP)에서 계산된 현재 주차가 유일한 트리거 입력. 사용자는 페이저로 보고 싶은 주차를 선택(읽기 전용 탐색)할 수 있다. EDD 변경 시(eddHistory append) 주차가 재계산돼 콘텐츠가 즉시 그 주차로 정렬된다.
- **한국 맥락**: NN주N일 + D-day 한국 관례 표기 필수. 마미톡·베이비빌리·280DAYS가 모두 채택한 '태아발달/엄마변화/아빠팁' 3블록은 한국 임신앱의 사실상 표준 골격이므로 동급 이상으로 갖춰야 신규 임산부 정복 가능. 아빠 팁은 280DAYS의 '아빠 어드바이스'처럼 부부 커뮤니케이션 도구로 포지셔닝 — 한국의 부부 공동 태교/태담 문화에 부합. 검진 안내(11~13주 NT, 16~18주 쿼드, 24~28주 임당)는 해당 주차 엄마변화/아빠팁 블록에 자연스럽게 녹여 'OO주에 OO검사 일정을 확인하세요'로 표기(이미 tip 데이터에 부분 반영됨).
- **엣지/주의**: ①pregnancy-weeks.json은 4~40주(37 entries)만 존재 → 4주 미만(착상 전후) 진입 시 '아직 콘텐츠 시작 전' 안내 + 4주 콘텐츠로 폴백, 40주 초과(예정일 경과) 시 40주 콘텐츠 유지 + '예정일이 지났어요, 의료진 안내를 따르세요' 배너. ②주차 경계(N주 6일→다음 주 0일) 자정 전환 시 콘텐츠가 끊김 없이 넘어가야 함. ③검진 안내가 이미 지난 주차(예: 14주에 12주 검사 안내)일 때 '시기가 지났다면 의료진과 상의'로 톤 조정. ④아빠 팁 블록은 파트너가 없는 사용자(미공유/한부모)에게도 의미 있도록 '함께하는 사람/나 자신을 위한' 중립 카피로 fallback. ⑤출산·유산·중절 등 임신 종료(outcomeType) 후에는 주차 콘텐츠 자동 중단(과거 주차는 아카이브에서만 열람).

### 데일리 팁 (매일 바뀌는 오늘의 한 가지)

- **무엇을**: 주차 단위 콘텐츠와 별개로, 하루 단위로 교체되는 짧은 팁. 같은 주차 안에서도 7일 동안 매일 다른 한 줄~두 줄 팁을 보여줘 '매일 앱을 열 이유'를 만든다. pregnancy-weeks.json의 주차 tip을 시드로 삼되, 같은 주 내 요일별 변주(영양/휴식/운동/마음챙김/검진 준비/태담/파트너 등 테마 로테이션)로 확장. 예: 16주 1일차 '오늘은 물을 한 컵 더 마셔보세요', 3일차 '편안한 자세로 5분 태담을 해보세요'. 각 팁은 카테고리 아이콘과 함께 1장의 작은 카드로 제시되고, 마음에 든 팁은 저장(북마크)할 수 있다.
- **어떻게 보이나**: 홈의 임신 카드 안 또는 바로 아래에 '오늘의 팁' 슬롯 1개(카드 1장). 탭하면 주차 콘텐츠 화면 상단의 '오늘의 팁' 영역으로 확장. 매일 0시 기준으로 자동 갱신, 날짜가 바뀌면 새 팁. 저장한 팁은 라이브러리의 '내가 저장한 글/팁' 모음에서 다시 본다. 광고·커머스 요소 없이 순수 정보 카드로만 구성.
- **입력**: 직접 입력 없음. 현재 주차 + 그 주차 안에서의 경과 일수(요일 인덱스)로 결정. 사용자 액션은 '저장(북마크)' 토글뿐. 알림(선택): '오늘의 팁이 도착했어요' 푸시는 영역 D(알림)와 연동되며 본 기능은 콘텐츠 표면만 정의.
- **한국 맥락**: 마미톡의 '매일매일 성장하는 태아 + 데일리 콘텐츠', 베이비빌리의 '매주 맞춤 꿀팁'을 일 단위로 더 잘게 쪼갠 형태 — 한국 임산부의 '매일 1회 앱 오픈' 습관(80% 사용률의 마미톡 패턴)을 겨냥. 팁 테마에 한국 맥락(국민행복카드 사용처 안내, 산모수첩 지참 리마인드, 임신부 금기·권장 음식)을 요일 로테이션에 포함.
- **엣지/주의**: ①같은 주에 머무는 7일을 다 채울 변주가 부족한 초저주차(4~5주)는 일반 임신 초기 팁 풀에서 보충. ②주차 전환일(주의 마지막 날→다음 주 첫날)에 데일리 팁과 주차 콘텐츠가 동시에 바뀌어 혼란스럽지 않도록 데일리는 '오늘', 주차는 '이번 주'로 라벨 명확화. ③기기 시간/타임존 변경으로 같은 날 두 번 갱신되지 않게 '하루 1회' 고정. ④40주 초과 시 데일리 팁은 '진통 신호 점검·병원 연락 준비' 위주 진통 맥락 팁으로 전환(disclaimerKey labor).

### 아기 크기 비교 (한국형 과일/사물 비유)

- **무엇을**: 이번 주 아기 크기를 친숙한 사물에 빗대 직관적으로 보여주는 콘텐츠. pregnancy-weeks.json의 fruitSize를 그대로 활용(4주 양귀비씨 → 40주 수박). '이번 주 아기는 OO만 해요' 한 줄 + 사물 아이콘/일러스트 + 실측 보조 정보(길이 cm·예상 체중 g)를 함께 표시. 비교 대상은 단순 텍스트가 아니라 시각 일러스트(사물 그림 옆에 태아 실루엣)로 크기감을 전달. 주차가 올라갈수록 사물이 커지는 누적 타임라인('지난 주: 사과 → 이번 주: 아보카도')도 제공해 성장 체감을 준다.
- **어떻게 보이나**: 주차 콘텐츠 화면 상단 히어로 영역(태아 일러스트 + D-day 옆/아래)에 '아기 크기' 한 줄과 사물 일러스트를 배치. 홈 임신 카드의 '크기 비유' 슬롯에도 축약 노출(IA 골격에 명시된 '크기 비유'). 탭하면 4~40주 크기 변화를 한눈에 보는 '성장 타임라인' 뷰로 확장(가로 스크롤, 현재 주차 하이라이트). 공유 기능으로 '우리 아기 이번 주 = 아보카도' 이미지 카드를 부부가 공유 가능.
- **입력**: 직접 입력 없음. 현재 주차가 fruitSize 매핑을 결정. 사용자는 타임라인에서 주차를 탐색(읽기 전용)하거나 크기 카드를 공유.
- **한국 맥락**: 현재 데이터(아보카도·그레이프프루트·허니듀 멜론·아이스버그 레터스·리크 등)는 서구 베이비센터 원본의 직역으로 한국 사용자에게 생소한 항목이 섞여 있음 → 한국에서 크기·친숙도가 통하는 비유로 현지화 권장: 콜리플라워→'양배추 한 통', 아이스버그 레터스→'양상추 한 통', 리크→'대파 한 단', 그레이프프루트→'자몽', 허니듀 멜론→'머스크멜론/참외' 등. 한국 임산부가 '아, 이만하구나' 바로 체감되는 농산물(참외·배·복숭아·옥수수·호박)로 우선 매핑하고, 애매한 항목은 의료감수자 확인 후 확정. 길이/체중은 한국 산부인과 통용 cm·g 표기.
- **엣지/주의**: ①fruitSize 문자열을 시각 일러스트로 1:1 매핑해야 하므로 일러스트 미보유 사물은 '크기만 글자' 폴백 금지 → 전 주차 일러스트 세트 확보 전까지는 통일된 사물 실루엣 + 라벨로 대체. ②동일 사물이 다른 주차에 중복 등장(22주 파파야, 28주에도 유사) 시 타임라인에서 '같은 크기로 보이는' 착시 방지 — 실측 cm/g를 보조로 항상 병기. ③성별/개월 편차로 실제 아기와 비교값이 다를 수 있으므로 '평균 추정치'임을 명시(백분위 의학 판단 텍스트 금지 규칙 준수 — 진단/이상 판정 카피 금지).

### 태아 발달 일러스트 (주차별 시각 자산)

- **무엇을**: 각 주차 태아의 모습을 보여주는 일러스트 세트. 주차 콘텐츠와 홈 임신 카드의 핵심 비주얼. 4~40주에 걸쳐 태아 형태 변화(꼬리 소실, 팔다리 형성, 얼굴 윤곽, 자세 변화 등 milestone과 연동)를 따뜻하고 의학적으로 오해 없는 톤의 일러스트로 표현. 일러스트는 ①홈 임신 카드 썸네일(작은 사이즈) ②주차 콘텐츠 히어로(큰 사이즈) ③크기 비교/타임라인의 태아 실루엣 세 곳에서 재사용. 주차 전환 시 일러스트가 부드럽게 바뀌어 성장의 시각적 보상을 준다.
- **어떻게 보이나**: 홈: 임신 카드 좌측/상단 태아 일러스트(IA 골격 명시 '태아 일러스트'). 주차 콘텐츠 화면: 상단 히어로에 큰 일러스트 + NN주N일 + D-day 오버레이. 라이트/다크 모드 양쪽에서 또렷해야 하며(위젯 WidgetColors adaptive 선례), 잠금화면/위젯의 D-day와 시각 일관성 유지. 접근성: 일러스트에 주차별 대체 텍스트('OO주 태아 일러스트') 제공.
- **입력**: 직접 입력 없음. 현재 주차가 일러스트 선택을 결정. 사용자 액션 없음(순수 표시 자산).
- **한국 맥락**: 마미톡·280DAYS의 '귀여운 태아 일러스트'가 핵심 정서적 차별화 요소 — 신규 임산부 정복을 위해선 일러스트 품질이 경쟁 동급 이상이어야 함. 한국 정서상 과도하게 사실적/의학 표본 같은 이미지보다 따뜻하고 친근한 톤 선호. 성별 미확정/비공개 정책과 충돌하지 않도록 중립적 태아 묘사(성기 묘사·성별 단정 금지).
- **엣지/주의**: ①37개 주차 전부 고유 일러스트를 일시에 확보하기 어려우면 발달 단계가 유사한 주차를 그룹핑(예: 4~7주, 8~12주)해 공유 일러스트로 시작하고 점진 고도화 — 단 '주차마다 다른 그림이 나온다'는 사용자 기대를 깨지 않도록 단계 라벨 병기. ②다크모드/잠금화면 대비 미달 시 또렷함 깨짐 → 라이트/다크 양쪽 검수 필수. ③초저주차(4주 양귀비씨급)는 형태가 거의 없어 일러스트보다 '세포 분열/착상' 추상 표현이 적절. ④접근성(VoiceOver) 대체텍스트 누락 시 시각장애 사용자에게 콘텐츠 공백.

### 전문가 감수 아티클 라이브러리

- **무엇을**: 주차별 단편 콘텐츠를 넘어, 산부인과 전문의 감수를 받은 심화 아티클을 주제별로 모은 읽을거리 라이브러리. 주제 카테고리 예: 산전검진 가이드(NT·쿼드·정밀초음파·임당 각각 '무슨 검사인지/언제/어떻게 준비'), 임신부 영양·금기 음식, 운동·체중 관리, 입덧·요통 등 증상 대처, 태교·태담, 출산 준비(출산가방 표준 목록, 진통 신호, 병원 가는 시점), 행정·복지(국민행복카드, 산모수첩, 출산휴가/육아휴직). 각 아티클은 '의료 감수: OOO 전문의' 표기 + 감수일자 + 면책 문구를 갖춘다. 주차 콘텐츠의 '관련 아티클 더 보기'에서 현재 주차에 적합한 글이 자동 추천돼 라이브러리로 연결된다.
- **어떻게 보이나**: 건강 탭 내 '임신 정보 라이브러리' 섹션. 진입 시 ①현재 주차 추천 아티클(상단 가로 카드) ②주제 카테고리 그리드 ③검색/저장한 글. 아티클 상세는 제목·감수자·읽는 시간·본문(소제목+이미지)·면책 문구·관련 글 구조. 데일리 팁/주차 블록에서 딥링크로 특정 아티클 진입. 저장(북마크)한 아티클은 '내 글 모음'에 적립. 광고·상품 링크·커머스 0 — 순수 정보만.
- **입력**: 직접 입력 없음(읽기 콘텐츠). 사용자 액션: 카테고리 선택, 키워드 검색, 아티클 저장/공유. 현재 주차가 추천 아티클 정렬에 영향(예: 11주엔 NT 검사 아티클 상단, 24주엔 임당 검사 아티클 상단).
- **한국 맥락**: 의료감수자(산부인과 전문의)·변호사 확보 전제이므로 '전문의 감수' 신뢰 배지가 마미톡('산부인과·소아과 전문의가 알려주는')과 동급 신뢰 신호 — 신규 임산부 유입의 핵심 설득 포인트. 산전검진 일정은 한국 표준(28주까지 4주 간격, 36주까지 2주, 이후 매주 / 11~13주 NT, 16~18주 쿼드, 24~28주 임당)을 정확히 반영. 행정 카테고리에 국민행복카드(임신·출산 진료비 바우처)·산모수첩·정부 지원 안내 포함이 한국 임산부에게 실질 가치. 출산가방 목록은 한국 산후조리원/병원 입원 기준 표준 항목으로 구성.
- **엣지/주의**: ①의료 정보 정확성 — 모든 아티클에 감수자/감수일/면책 필수, 진단·처방·복약 지시 톤 금지(정보 제공만). pregnancy-weeks.json 본문도 현재 'agent 자동 생성'이라 의료 검증 미완(Active TODO H-4) → 라이브러리 콘텐츠는 출시 전 전문의 검수 게이트 필수. ②검색 결과 0건 시 빈 화면 대신 인기/추천 아티클 노출. ③감수 후 의학 지침 변경(예: 검진 권고 개정) 시 콘텐츠 갱신·재감수 운영 흐름 필요(감수일 노출로 사용자에게 최신성 신호). ④법무 검토 영역(개인정보·면책 범위)은 변호사 확보됨 — 면책 문구 카피는 법무 확정본 사용. ⑤커머스 0 원칙상 출산가방/용품 아티클에서 특정 브랜드·구매 링크 삽입 금지(품목 일반명만).

## C. 산전검진·건강 (Prenatal Checkups & Health) — 한국 임산부 산전관리 정복

### 산전검진 통합 기록 (일정+결과수치+초음파사진 한 객체)

- **무엇을**: 한 번의 병원 방문을 '하나의 검진 카드'로 묶어 기록한다. 한 카드 안에 (1) 방문 정보: 검진 날짜, 그날의 NN주N일/D-day 자동 표기, 병원명, 담당의, 검진 유형(정기/NT/쿼드/정밀초음파/임당 등 한국 검진 라벨 선택), (2) 그날 측정한 결과 수치: 산모 혈압, 산모 체중, 태아 추정체중(g), 태아심박수(bpm), 자궁저 높이, 양수지수, 소변(단백뇨/당뇨) 등 한국 산모수첩 정기진찰 항목을 그대로 칸으로 제공(해당 없는 칸은 비워둠), (3) 그날 받은 초음파 사진/영상을 여러 장 첨부, (4) 다음 검진 예약일과 자유 메모를 담는다. 검진을 저장하면 홈 케어 카드의 빠른 액션과 캘린더에 다음 예약일이 자동으로 떠서 '받은 검진'과 '받을 검진'이 한 흐름으로 이어진다. 수치는 추세로 쌓여 혈압·체중·태아체중이 회차별로 어떻게 변하는지 한눈에 본다(의학적 정상/비정상 판정 문구는 절대 넣지 않고 '내 기록 추세'만 보여줌).
- **어떻게 보이나**: 건강 탭 안 '산전검진' 섹션 = 회차 리스트(최신순, 각 행에 날짜+NN주N일+검진유형 칩+썸네일 초음파). 행을 누르면 상세(수치 표 + 초음파 갤러리 + 메모). 신규 기록은 ➕기록 탭의 임신 분기에서 '병원 검진' 진입. 초음파는 가로 스와이프 갤러리 + 핀치 확대. 수치 추세는 건강 탭 상단 미니 라인(체중/혈압/태아체중 토글). 입력은 숫자 키패드 + 단위 고정 라벨로 오입력 최소화.
- **입력**: 사용자 입력: 검진 날짜(기본 오늘), 검진 유형 선택, 병원/담당의(자동완성+직전값 기억), 수치 칸(선택 입력, 빈칸 허용), 초음파 사진(카메라/앨범 다중 선택), 메모, 다음 예약일. NN주N일/D-day는 등록된 출산예정일 기준 자동 계산. 단위는 시스템 고정(kg/mmHg/g/bpm/cm).
- **한국 맥락**: 한국 산모수첩 '정기 진찰' 기록 항목(혈압·체중·체질량지수·소변 단백뇨/당뇨·부종·태아심음·자궁저)을 칸 구조로 그대로 반영해 '디지털 산모수첩'처럼 쓰이게 한다. NN주N일+D-day 표기는 한국 임신앱 표준. 초음파 사진은 한국 부모가 가장 애착하는 자산이라 검진과 한 묶음으로 보관하는 것이 핵심 가치. 검진 유형 라벨은 한국 실제 명칭(NT·쿼드·정밀초음파·임당) 사용.
- **엣지/주의**: 출산예정일 미등록 시 NN주N일 자동표기 자리에 '예정일 등록하면 주차가 표시돼요' 플레이스홀더. 수치 전 칸 비워도 저장 가능(초음파만 올리는 경우). 예정일이 eddHistory로 갱신되면 과거 검진의 NN주N일 표기는 '그 검진 시점의 당시 주차'로 고정(소급 변경 금지)해 기록 신뢰성 유지. 초음파 다량 첨부 시 업로드 진행상태 표시, 오프라인이면 큐잉 후 자동 동기화 안내. 의학 면책 문구 상시 노출. 다태아(쌍둥이) 시 태아별 추정체중 칸 분리. 비정상 수치라도 앱이 '위험/정상' 판정하지 않고 '담당의와 상담' 안내만.

### 주차별 산전검진 체크리스트 (NT/쿼드/정밀초음파/임당 권장 시기 타임라인)

- **무엇을**: 한국 표준 산전검진 일정을 주차 타임라인으로 미리 깔아주는 가이드형 체크리스트. 보유 자산인 기존 체크리스트를 '검진 전용' 항목으로 확장해, 등록된 출산예정일 기준으로 각 검진의 권장 시기 창(window)에 도달하면 해당 항목이 '지금이 시기예요'로 활성화된다. 기본 탑재 항목: 첫 산전진찰/심장박동 확인(6~8주), 초기 정밀+NT 목덜미투명대(11~13주6일), NIPT/통합·쿼드 기형아 선별(쿼드 15~20주), 정밀(상세) 초음파(20~24주), 임신성당뇨 선별검사(24~28주), 백일해/독감 예방접종 시기, 빈혈·후기검사(28주 이후), GBS 검사(35~37주), 만삭 정기검진(28주까지 4주·36주까지 2주·이후 매주 리듬 안내) 등. 각 항목에는 '왜 이 시기에 하는지' 한 줄 의료감수 설명과 '예약 잡기/검진 기록하기' 바로가기가 붙는다. 항목을 체크하면 위의 '산전검진 통합 기록'으로 연결돼 결과를 바로 남길 수 있다. 사용자는 항목 추가/숨김도 가능.
- **어떻게 보이나**: 임신 건강 영역의 '검진 체크리스트' 섹션 = 주차 그룹 헤더(초기/중기/후기) 아래 카드 리스트. 각 카드: 검진명 + 권장 시기 배지(예: 11~13주) + 상태(다가옴/지금 시기/완료/지남) + 한 줄 설명. 현재 주차에 해당하는 항목은 상단 고정 + 강조 색. 완료 시 체크+완료일 표기. 홈 케어 카드 빠른 액션에 '이번 검진' 1건을 노출해 진입 단축.
- **입력**: 자동: 출산예정일 기준 각 항목 권장 시기 창 계산 → 도달 시 활성화. 사용자 입력: 체크(완료 처리, 완료일 기록)/스킵·숨김/직접 항목 추가(제목·목표 주차). 항목에서 '검진 기록하기' 누르면 통합 기록 작성으로 이동. 알림 받기 토글 시 권장 시기 진입 며칠 전 리마인드.
- **한국 맥락**: 시기·검출 특성은 한국 표준 출처 반영: NT 11~13주6일(초기 정밀과 함께), 쿼드 임신중기 15~20주, NIPT 10주 이후 가능, 정밀(상세)초음파 20~24주(장기 형성+양수 풍부해 기형 발견 최적기), 임당 24~28주, 정기진찰 리듬 28주까지 4주/36주까지 2주/이후 매주. 더블마커(9~13주)·트리플(16~18주) 등 선택검사도 참고 항목으로 제공하되 '병원·산모 상황별로 다름' 명시.
- **엣지/주의**: 예정일 미등록이면 시기 자동활성화 불가 → '예정일을 등록하면 검진 시기를 맞춰드려요' 유도. 예정일 변경(eddHistory) 시 모든 미완료 항목의 권장 시기 창을 재계산하되 이미 완료한 항목의 완료일은 보존. 권장 시기를 지난 미완료 항목은 삭제하지 않고 '시기 지남(병원과 상담)'으로 부드럽게 표기(불안 유발 문구 금지). NIPT처럼 모든 산모가 받지 않는 선택검사는 '선택' 태그로 구분해 미완료를 결함처럼 보이지 않게 함. 의료 감수 설명은 감수자 검토 전까지 일반 안내 톤 유지, 진단·지시 문구 금지.

### 한국 산전검진·정부 바우처·산모수첩 매핑 (국민행복카드 잔액 흐름 + 디지털 산모수첩) — 최대 차별화

- **무엇을**: 경쟁 앱이 0/11로 비어 있는 한국형 행정 동행 기능. 세 축을 하나로 묶는다. (1) 국민행복카드 임신·출산 진료비 바우처 트래커: 지원금 총액(임신 1회 100만원, 다태아 140만원, 분만취약지 +20만원)을 시작 잔액으로 등록하고, 검진/약제비 사용액을 기록할 때마다 남은 잔액과 사용 내역이 쌓인다. 사용 가능 기한(출산예정일 이후 일정 기간까지)을 D-day로 알려 '소멸 전 잔액'을 챙기게 한다. (2) 산모수첩 매핑: 위의 검진 통합 기록을 한국 산모수첩 항목 구조(산모 건강기록·정기진찰·검사 소견)로 정렬해 '내 디지털 산모수첩' 한 화면으로 보여주고, 병원 갈 때 보여줄 수 있는 요약 형태로 정리한다. (3) 검진↔바우처↔수첩 연결: 검진을 기록하면 '이 검진 비용을 바우처에서 사용했나요?'를 물어 잔액 차감과 산모수첩 기록을 한 번에 처리. 신청 경로(정부24 맘편한임신/건강보험공단)와 발급·신청 안내 카드도 제공(앱은 안내만, 외부 결제·발급 대행 없음 → 커머스0 유지).
- **어떻게 보이나**: 건강 탭 '임신 행정' 섹션 또는 설정 내 '국민행복카드/산모수첩' 진입. 바우처 화면: 큰 잔액 링(남은 금액) + 사용 기한 D-day + 사용 내역 리스트(날짜·항목·금액·연결된 검진). 산모수첩 화면: 산모 건강기록(혈액형·키·기저질환 등 1회 입력) + 회차별 정기진찰 표 + 검사 소견 타임라인을 한 스크롤로. '신청 방법' 안내는 접이식 카드(외부 링크는 시스템 브라우저로 안내).
- **입력**: 사용자 입력: 바우처 시작 금액(단태/다태아 프리셋 또는 직접), 사용 기한(예정일 기준 자동 계산 + 수정 가능), 사용 내역(검진 기록과 연동해 1탭 차감 또는 수동 추가). 산모수첩: 산모 기본 건강정보 1회 입력(키·혈액형·기저질환·알레르기), 나머지는 검진 기록에서 자동 채움. 신청 안내는 읽기 전용.
- **한국 맥락**: 수치·정책은 한국 2026 기준 반영: 바우처 임신 1회 100만원, 다태아 140만원 기본, 분만취약지 +20만원(단태 120·다태 160), 다태아 태아당 100만원 되도록 추가, 사용기한은 분만예정일(출산일) 이후 일정 기간(2세 미만 영유아 진료비까지 포함). 신청 경로=정부24 '맘편한임신' 원스톱/건강보험공단 홈페이지. 산모수첩=임신 6~8주 심박 확인 후 발급되는 한국 표준 모자보건수첩 구조(산모 건강기록+정기진찰+검사소견). 이 매핑이 '신규 임산부 정복'의 핵심 차별점.
- **엣지/주의**: 정책 금액은 변동 가능 → 하드코딩처럼 보이지 않게 '제도 기준 안내(변경될 수 있음, 공단 확인)' 면책 + 프리셋은 수정 가능하게. 바우처는 산모 본인부담금에만 쓰이고 비급여 일부는 대상 외일 수 있음을 안내(앱이 자동 판정 금지). 다태아·분만취약지 추가지원은 옵션 토글로 단순화. 외부 신청·결제는 앱 밖이며 앱은 어떤 금전 처리도 하지 않음(광고0·커머스0 가치 보존). 사용 기한 임박 시 알림. 잔액 음수 방지(사용액이 잔액 초과 시 경고). 산모수첩 요약은 의료기록 대체가 아님을 명시(공식 기록은 병원 발급본).

### 진료 준비 질문 가이드 (검진 전 궁금증 정리 → 검진 기록으로 이어짐)

- **무엇을**: 짧은 진료 시간에 묻고 싶은 걸 놓치지 않도록, 검진 전에 질문을 모아두는 가이드. 현재 주차/다가오는 검진 유형에 맞춘 추천 질문 풀을 제공한다(예: NT 검진 전 '결과는 언제 나오나요/추가검사가 필요한가요', 임당 검사 전 '검사 전 금식은 어떻게/물은 마셔도 되나요', 후기 '태동이 줄면 언제 연락해야 하나요/유도분만 기준은'). 사용자는 추천 질문을 탭으로 담거나 직접 질문을 추가해 '이번 검진 질문 목록'을 만든다. 검진 당일 이 목록을 병원에서 펼쳐 보고, 받은 답변을 각 질문 옆에 바로 받아적으면 그대로 해당 검진의 통합 기록 메모로 저장된다. 증상/걱정거리(예: 부종·출혈·심한 입덧)도 질문으로 빠르게 담아 '말할 것'을 잊지 않게 한다.
- **어떻게 보이나**: 검진 상세 또는 체크리스트 항목 안의 '진료 질문' 탭. 상단=추천 질문 칩(주차·검진유형 맞춤, 탭하면 목록에 추가), 하단=내 질문 목록(체크박스로 '물어봄' 표시 + 답변 받아쓰기 인라인). 검진 당일엔 홈 케어 카드 빠른 액션 또는 캘린더 예약 항목에서 '오늘 질문 보기'로 바로 진입. 큰 글씨 '진료실 모드'(병원에서 한 손으로 보기 좋은 리스트 뷰) 제공.
- **입력**: 사용자 입력: 추천 질문 선택/직접 질문 작성/순서 변경/삭제, 진료 중 답변 받아쓰기, 증상 메모 추가. 자동: 현재 주차+다가오는 검진 유형에 맞춰 추천 질문 풀 노출. 답변을 저장하면 해당 검진 통합 기록의 메모로 연결.
- **한국 맥락**: 추천 질문은 한국 산전검진 흐름에 맞춤(NT/쿼드/정밀초음파/임당/후기검진별). 한국 진료가 짧고 빠른 점을 배려해 '미리 정리→현장에서 빠르게 확인'에 최적화. 부부공유 자산과 연계해 배우자도 질문을 함께 담을 수 있게(태담·동행 문화 반영). 의료 면책: 추천 질문은 참고용이며 진단·치료 지시가 아님 명시.
- **엣지/주의**: 추천 질문은 어디까지나 '대화 보조'이며 앱이 의학적 답을 주지 않음(답변은 사용자가 의사에게 들은 내용을 적는 것). 응급 신호(출혈·심한 복통·태동 급감 등)를 질문으로 담으면 '응급은 질문 미루지 말고 즉시 병원 연락' 안내 배너 노출. 검진이 끝나 답변까지 적힌 질문 목록은 통합 기록에 귀속되어 회차별로 다시 열람 가능. 답변 미입력 항목은 다음 검진으로 '이월하기' 옵션. 부부공유 시 작성자 표시. 어떤 텍스트도 상업 링크·광고 미포함.

## 영역 D. 정서·기록 (임신 일기 / 만삭사진 타임랩스 앨범 / 초음파 주차순 타임라인 / 아기에게 쓰는 편지·태담)

### 임신 일기 (주차 자동 태깅 + 사진 첨부)

- **무엇을**: 임산부가 그날의 몸 상태·기분·먹은 것·태동·생각을 자유롭게 기록하는 일기. 모든 글에 작성 당시의 NN주N일과 D-day가 자동으로 새겨져, 나중에 '몇 주에 무슨 일이 있었지'를 시간이 아니라 임신 주차로 되짚을 수 있다. 사진(입덧 중 겨우 먹은 죽 한 그릇, 처음 산 배냇저고리, 부은 발 등) 여러 장을 첨부한다. 기분은 한 번의 탭으로 고르는 이모지/칩(설렘·불안·벅참·힘듦·평온 등)으로 가볍게 남기고, 본문은 쓰고 싶을 때만 길게 쓴다. 입덧·두통·허리통증 같은 그날의 컨디션을 가벼운 태그로 붙일 수 있어, 글을 안 써도 '오늘 어땠는지'가 남는다. 출산 후 육아 일기와 같은 줄기로 이어져, 한 아이의 '뱃속부터 지금까지'가 끊김 없는 하나의 기록 자산이 된다.
- **어떻게 보이나**: 기록(➕) 탭에서 '임신 일기' 진입 → 작성. 열람은 캘린더 탭(날짜에 일기 점/썸네일)과 건강 탭의 '임신 일기' 묶음 두 경로. 일기 카드는 큰 사진 + 'NN주N일 · D-day' 배지 + 기분 이모지 + 본문 미리보기. 홈의 임신 카드 빠른 액션에 '오늘 일기' 버튼을 두어 한 번에 진입. 주차 헤더로 그룹핑된 세로 피드(예: '23주' 아래 그날들). 출산 후에는 같은 일기 피드가 자연스럽게 육아 일기로 이어지며, 임신 구간은 '임신 시절' 섹션으로 접근 유지.
- **입력**: 본문 텍스트(선택, 미작성 허용) / 기분 칩 1개 이상 탭 / 컨디션 태그 다중 선택(입덧·두통·부종·불면·요통·기타) / 사진 0~여러 장 첨부(갤러리·카메라) / 날짜(기본 오늘, 과거 소급 입력 가능) / 주차·D-day는 등록된 마지막 생리일/출산예정일 기준 자동 계산되어 사용자 입력 불필요.
- **한국 맥락**: 글의 시간축을 '날짜'가 아니라 'NN주N일 + D-day'로 보여주는 것이 한국 임산부 멘탈모델의 핵심(다들 '지금 몇 주냐'로 대화함). 컨디션 태그는 산모수첩에 적어 두던 '주차별 증상 메모' 습관을 디지털화한 것. 기분 기록을 의료적 우울 스크리닝으로 포장하지 말 것(가벼운 정서 기록에 머무름).
- **엣지/주의**: 출산예정일 변경(eddHistory append) 시 과거 일기의 주차 라벨이 흔들릴 수 있음 → 각 일기는 '작성 시점에 계산된 주차'를 고정 표기하고, EDD 보정으로 재계산하지 않음(추억의 사실 보존). 임신 종료(유산·사산)가 기록된 경우 일기 피드는 절대 삭제·숨김하지 않고 그대로 보존(데이터 삭제 금지 룰). 사진 0장+본문 0자+기분만 있는 '초경량 일기'도 유효 저장. 오프라인 작성 시 큐잉 후 자동 동기화. 미래 날짜 입력은 오늘로 클램프.

### 배(만삭) 사진 타임랩스 앨범

- **무엇을**: 같은 구도로 옆모습 배 사진을 주차마다 한 장씩 모아, 점점 불러오는 배의 변화를 한 흐름으로 보여주는 전용 앨범. 각 사진엔 'NN주N일'이 자동으로 박혀, 시간순으로 넘기면 배가 자라는 과정이 보이고, 끝에서 '타임랩스 재생'을 누르면 4주→만삭까지가 부드럽게 이어지는 짧은 영상처럼 흘러간다. 초반(배가 안 보이는 시기)부터 찍어 두면 대비가 극적이라, 매주 한 컷 습관을 만들도록 가볍게 권한다. 만삭 구간(28~34주)에 들어서면 '지금이 만삭사진 황금기'라고 살짝 알려준다.
- **어떻게 보이나**: 건강 탭의 '배 사진 앨범' 또는 임신 일기 안의 별도 탭. 메인은 가로 스크롤 카드 스택(주차 라벨 오버레이) + 상단 '타임랩스 재생' 버튼. 격자(그리드) 보기로 전환하면 4주·5주·…·40주 슬롯이 한눈에 보이고, 빈 주차는 '이 주 사진 추가' placeholder. 촬영 보조로 직전 주 사진을 반투명 가이드(고스트 오버레이)로 겹쳐 보여줘 같은 구도 유지. 만삭 황금기(28~34주) 진입 시 홈 임신 카드/앨범 상단에 부드러운 안내 띠.
- **입력**: 사진 첨부(카메라 직접 촬영 권장, 갤러리 선택 가능) / 주차는 촬영일 기준 자동, 사용자가 슬롯을 수동 지정/이동 가능 / 한 주차에 여러 장 찍었으면 대표 1장 선택 / 타임랩스 재생 속도(느리게/보통/빠르게) 선택 / 만든 타임랩스/대표 사진을 가족에게 공유하거나 기기에 저장.
- **한국 맥락**: 만삭사진은 한국에서 결혼·돌 사진급 통과의례가 됐고, 스튜디오 대신 '셀프 홈 만삭사진'이 빠르게 느는 흐름을 정면으로 받는다. 황금기 안내(28~34주, 베스트 28~32주)는 실제 촬영작가 가이드와 일치. 단, 스튜디오 예약·상품 추천 같은 커머스 요소는 절대 넣지 않음(광고0·커머스0). '배를 드러내 강조하는' 요즘 문화를 존중하되, 노출 강요는 하지 않고 구도/실루엣은 사용자 자유.
- **엣지/주의**: 구도가 매주 달라 타임랩스가 들쭉날쭉할 수 있음 → 고스트 가이드 + 수동 위치/확대 보정으로 완화하되 자동 얼굴/배 정렬은 과약속하지 않음(보정 실패 가능성 봉쇄). 사진은 민감 신체 이미지이므로 가족 공유 외 외부 노출/분석에 절대 쓰지 않음. 임신 종료 시에도 앨범 보존(삭제 금지). 주차 4주 미만/40주 초과 등 범위 밖 촬영도 '기타' 슬롯으로 수용. 타임랩스는 기기 내 생성/재생만(서버 영상 처리·과약속 금지).

### 초음파 주차순 타임라인

- **무엇을**: 병원에서 받은 초음파 사진을 찍은 날짜가 아니라 임신 주차 순서로 자동 정렬해, 콩알만 하던 태아가 사람 모습을 갖춰가는 과정을 한 줄기로 보여주는 타임라인. 각 초음파에 'NN주N일 + 그 주의 태아 크기 비유(예: 라임만 해요)'와 한 줄 메모(예: 첫 심장소리 들음)를 붙인다. 한국 산전검진의 결정적 장면들(11~13주 목투명대 NT, 16주 쿼드, 20주 정밀초음파, 24~28주 입체초음파)을 타임라인에 마일스톤으로 표시해, '다음에 어떤 초음파를 보게 되는지'를 미리 알려준다. 출산 후 아기 사진첩 맨 앞에 '뱃속 첫 사진들'로 이어 붙는다.
- **어떻게 보이나**: 건강 탭의 '초음파' 묶음. 세로 타임라인(위→아래로 주차 증가), 각 노드 = 초음파 썸네일 + 주차 배지 + 크기 비유 + 메모. 정밀초음파·NT 같은 주요 검사 주차에는 색이 다른 마일스톤 마커. 검진 일정(캘린더의 산전검진 일정과 연동)과 같은 주차 선상에 놓여, '20주 정밀초음파' 검진과 그날 받은 초음파 사진이 한 자리에서 보인다. 사진 탭하면 원본 크게 보기 + 핀치 확대. 출산 후 baby 사진첩 진입 시 맨 앞 섹션으로 '임신 중 초음파' 노출.
- **입력**: 초음파 사진 첨부(촬영/갤러리) / 촬영일 입력(기본 오늘, 과거 소급) → 주차 자동 / 한 줄 메모(선택) / 검사 종류 태그(선택: 일반·NT·정밀·입체·기타) / 같은 날 여러 장 첨부 가능. 사진 속 측정 수치(체중·머리둘레 등) 자동 인식(OCR)은 하지 않으며 사용자가 원하면 메모에 직접 적음.
- **한국 맥락**: 주차별 크기 비유 텍스트는 보유 자산 pregnancy-weeks.json(4~40주 37개 엔트리, 양귀비씨→…)을 그대로 활용. 산전검진 결정적 일정(NT 11~13주·쿼드 16주·정밀초음파 20주·입체 20~28주)은 한국 표준 일정과 일치. 경쟁 앱들이 초음파 수치 OCR/AI 성별예측을 내세우지만, BabyCare는 의료 수치 자동판독·성별 예측 같은 과한 약속을 하지 않음(면책·정확성 리스크 회피, AI 가드레일 준수).
- **엣지/주의**: 초음파는 민감 의료영상 → Analytics/Crashlytics 등 어떤 진단 페이로드에도 절대 포함 금지. 크기 비유나 검진 일정에 의학적 판단 텍스트(정상/이상 등) 금지, 면책 문구 동반. 출산예정일 보정으로 과거 초음파 주차 라벨이 바뀌면 혼란 → 일기와 동일하게 '촬영 시점 주차'를 고정 표기. 같은 주차에 사진 여러 장이면 타임라인에 묶어서(캐러셀) 표시. 임신 종료 시에도 초음파 보존.

### 아기에게 쓰는 편지 · 태담

- **무엇을**: 엄마·아빠가 태명을 부르며 뱃속 아기에게 직접 말을 거는 편지/태담 기록. '○○(태명)아, 오늘 처음 발로 찼지' 같은 짧은 말부터, 출산 후 아이가 크면 보여줄 긴 편지까지 모은다. 각 편지에도 NN주N일이 새겨져, 주차별로 아기에게 어떤 마음이었는지가 남는다. 태담을 거들도록 '오늘의 태담 한마디'(아침 인사·식사 묘사·잠들기 전 인사 등 가벼운 제안)를 매일 가볍게 띄워, 무엇을 말해야 할지 막막한 부모를 돕는다. 부부가 함께 쓰는 임신이면 둘 다 편지를 남길 수 있어, 아기는 나중에 '엄마 편지'와 '아빠 편지'를 모두 받는다. 출산 후 아이 프로필에 '뱃속에서 받은 편지함'으로 이어져, 첫 생일·먼 훗날 열어 줄 선물 자산이 된다.
- **어떻게 보이나**: 기록(➕) 탭의 '아기에게 편지/태담' 또는 임신 일기 내 별도 탭. 편지지 느낌의 작성 화면(태명 호명이 자동으로 머리말에 들어감). 모아 보기는 주차순 카드 피드 + 작성자(엄마/아빠) 라벨. 홈 임신 카드의 빠른 액션에 '오늘 태담'을 두어 1탭 진입, 매일 '오늘의 태담 한마디' 제안 칩을 가볍게 노출(누르면 그 문장으로 작성 시작). 사진·짧은 음성 메모(아기에게 들려주듯 녹음) 첨부 가능. 출산 후 baby 프로필에서 '뱃속에서 받은 편지함' 섹션으로 통째 이어짐.
- **입력**: 편지 본문 텍스트 / 태명(임신 등록 시 입력된 태명 자동 사용, 없으면 입력 유도) / 작성자 선택(엄마/아빠 등, 부부 공유 시) / 선택적 사진·음성 메모 첨부 / 공개 시점 메모(선택: '돌에 보여주기' 같은 자유 메모, 시스템 잠금 기능 아님) / 주차·D-day 자동. 매일 제안되는 '태담 한마디'는 탭 한 번으로 본문 시드로 채워짐.
- **한국 맥락**: 태담(胎談)은 태명을 부르며 부드럽고 또렷한 어조로 말 걸고 배를 쓰다듬는 한국 고유의 태교 문화 → 태명 자동 호명·'오늘의 태담 한마디' 제안이 이 의식을 자연스럽게 거든다. '나중에 아이에게 보여줄 편지'는 한국 태교일기/편지 문화의 핵심 정서(아이에게 주는 선물). 부부 공동 작성은 보유 자산인 부부 공유 기능과 결이 맞음. 제안 문구는 의료·종교 색을 빼고 일상적·따뜻한 톤으로 한정.
- **엣지/주의**: 편지는 매우 사적·정서적 → 임신 종료(유산·사산) 시 절대 삭제·자동숨김하지 않으며, 사용자가 직접 보지 않기로 선택할 때만 접힘(데이터 삭제 금지 룰 엄수). '공개 시점 메모'는 추억용 자유 텍스트일 뿐 실제 시간잠금/암호 기능으로 과약속하지 않음. 부부 공유 시 상대가 쓴 편지는 읽기 가능하되 편집은 작성자 본인만(공유 데이터 경로 권한 준수). 음성 메모는 민감 데이터로 외부 분석/Analytics에 절대 사용 금지. 태명 미입력 상태에서도 '아기에게' 같은 기본 호명으로 작성 가능.

## 영역 E. 도구 — 예정일 계산기 / 출산준비물 체크리스트 / 출산가방 체크리스트 / 출산계획서 / 아기 이름 짓기 / 홈 위젯 / 쌍둥이·다태아 모드

### 예정일 계산기 (LMP / 초음파 / IVF 3-방식)

- **무엇을**: 임신 등록 시 또는 설정에서 언제든 호출 가능한 출산예정일·현재 주차 계산 도구. 세 가지 입력 방식을 탭으로 제공한다. ① LMP(마지막 생리 시작일): 입력일 + 280일(40주), 한국 산부인과 표준인 네겔레 공식(생리 시작월 -3 또는 +9, 일 +7)을 그대로 적용. ② 초음파 보정: 검진일 + 그날 측정된 CRL(머리엉덩길이, mm) 또는 의사가 알려준 'NN주 N일'을 입력하면 역산해 예정일을 보정. ③ IVF(시험관): 이식일 + 배아 일수(3일배아/5일배아 선택)로 정확 계산(3일배아=이식일이 임신 3주2일, 5일배아=3주4일 기준). 계산 결과로 '출산예정일(D-day) + 현재 NN주 N일 + 임신 N개월차 + 삼분기(1·2·3기)'를 한 화면에 요약한다.
- **어떻게 보이나**: 임신 등록 플로우(첫 진입) 안의 1스텝, 그리고 설정 > '임신 정보 수정'에서 재진입. 세 방식은 상단 세그먼트 탭(생리일 / 초음파 / 시험관). 입력 즉시 하단 결과 카드가 실시간 갱신(D-day 큰 숫자 + 'NN주 N일' + 예정일 날짜). 결과 확정 시 홈 케어 프로필 카드 스택의 임신 카드와 위젯에 즉시 반영. 초음파 보정 시 LMP 예정일과 며칠 차이 나는지 작게 비교 표기('생리일 기준보다 5일 늦음').
- **입력**: 방식별 입력값: ①LMP=날짜피커(달력). ②초음파=검진일 날짜피커 + CRL 숫자 입력(선택) 또는 '의사가 말한 주차' NN주 N일 스텝퍼. ③IVF=이식일 날짜피커 + 배아일수 토글(3일/5일). 출력=예정일 Date, 현재 주차(NN주N일), 개월수, 삼분기, (보정 시)LMP대비 차이일수.
- **한국 맥락**: 한국 산부인과는 LMP 280일/네겔레가 1차 기준이나 초기 초음파 CRL 보정을 표준으로 우선시함 → 앱도 '초음파가 있으면 그 값을 권장'한다는 안내 문구. '의사가 알려준 주차'를 직접 넣는 경로가 한국 임산부에게 가장 직관적(병원에서 주차로 듣고 옴). 결과는 반드시 NN주N일+D-day 한국 표기. 의료 면책 문구 필수('정확한 예정일은 담당 의료진 진단 기준').
- **엣지/주의**: 예정일은 진료 중 자주 바뀜 → 덮어쓰기가 아니라 변경 이력으로 누적(이전 예정일과 비교 보여줌, 마지막 확정값을 현재값으로). 미래 날짜·과거 1년 초과 LMP 입력 시 경고. CRL 권장 범위(약 7~13주, 45~84mm) 벗어나면 '초기 초음파에서만 정확' 안내. 윤년/월말 날짜 경계는 일 단위 계산으로 안전 처리. 세 방식 간 결과 차이가 2주 이상이면 입력 재확인 유도.

### 출산준비물 체크리스트 (아기·산모 용품)

- **무엇을**: 출산 전 집에 미리 갖춰야 할 물품을 카테고리별로 체크하는 리스트(=출산가방과 별개, '집에 사둘 것'). 의료감수 기반 한국 표준 번들 항목을 기본 제공: 신생아 의류(배냇저고리·우주복·손싸개/발싸개·모자), 수유(젖병·젖꼭지·소독기·분유 또는 수유용품·수유쿠션), 위생(기저귀 신생아용·물티슈·가제손수건 多·아기세제), 목욕/케어(아기욕조·아기로션·체온계·손톱가위·면봉), 수면(아기침대/범퍼·속싸개·방수요), 외출(바구니카시트·아기띠), 산모 회복용품(수유브라·복대·산후패드·수유패드)을 묶음으로 담아둔다. 사용자는 체크/추가/삭제하고 진행률(%)을 본다.
- **어떻게 보이나**: 건강 탭 > '임신' 영역 안의 '출산 준비' 섹션, 또는 홈 임신 카드의 빠른 액션 '준비물'로 진입. 카테고리 접이식 그룹 + 각 항목 체크박스 + 진행률 링/바. 항목 길게 눌러 메모(브랜드·구매처 메모는 자유 텍스트, 커머스 링크 없음). 사용자 직접 항목 추가 가능. 임신 3기(28주~) 진입 시 홈 카드에서 '준비물 챙길 시기예요' 부드러운 리마인드(광고 아님).
- **입력**: 기본 번들 항목(카테고리·항목명·권장 시작 주차)은 의료감수 JSON 자산으로 내장. 사용자 입력=체크 토글, 커스텀 항목 추가(이름), 항목별 자유 메모. 출력=카테고리별/전체 진행률.
- **한국 맥락**: 한국 표준: 배냇저고리·속싸개·가제손수건은 다량(가제손수건 20~30장), 바구니카시트는 퇴원 시 필수(법규+병원 안내). 산모 회복용품(산후패드·수유패드·복대)을 신생아용과 분리해 보여주는 게 한국 출산 문화에 맞음. 보유 자산 prenatal-checklist.json의 'postpartum_prep / 아기 용품 구매 리스트' 항목을 이 도구로 확장. 커머스 0 원칙 → 추천/구매 링크 절대 없음, 순수 체크 도구.
- **엣지/주의**: 쌍둥이·다태아 모드일 때 수량 안내 자동 2배(예: 카시트 2개, 침대 2개) 문구. 모유수유 계획/분유 계획에 따라 수유 항목 분기(둘 다 켜두고 사용자가 선택). 항목 0개 삭제 시 빈 상태 안내 + '기본 목록 다시 불러오기'. 이미 출산(born)으로 전환된 임신은 리스트를 읽기전용 아카이브로 보존(삭제 금지).

### 출산가방 체크리스트 (병원용 / 산후조리원용 분리)

- **무엇을**: 출산 임박 시(보통 34~36주) 챙기는 '들고 갈 가방' 전용 체크리스트. 한국 현실 반영해 ①병원 입원가방(2~4일)과 ②산후조리원 가방(2~3주)을 별도 탭으로 분리한다. 의료감수 한국 표준 목록 내장 — 공통 필수: 신분증·산모수첩·산모수첩(국민행복카드)·진료카드·입원 서류. 병원용 산모: 수유복/앞트임 상하의 2~3벌, 산후패드, 산후팬티 5~6장, 수유브라, 세면도구, 슬리퍼, 휴대폰 충전기(긴 케이블), 물병/빨대컵. 병원용 아기: 퇴원복(배냇/우주복), 속싸개, 모자, 손싸개, 바구니카시트(퇴원 차량용). 보호자(남편)용: 갈아입을 옷, 세면도구, 간단 식비/카드, 카메라. 산후조리원용은 양·기간을 늘린 별도 목록(수유복 多, 개인 세면·기초화장품, 수유쿠션, 노트북/이어폰 등).
- **어떻게 보이나**: 출산준비물 체크리스트와 같은 '출산 준비' 섹션 내 별도 항목으로 분리(혼동 방지). 상단 세그먼트 탭: 병원 가방 / 조리원 가방 / 보호자. 각 탭 카테고리 접이식 + 체크 + 진행률. 34주 진입 시 홈 임신 카드에서 '가방 미리 싸두기' 리마인드. 가방 위치 메모 칸('현관 신발장 위' — 가족 공유 시 배우자도 봄).
- **입력**: 병원/조리원/보호자 3목록 의료감수 JSON 자산 내장(항목명·권장 챙김 시기). 사용자=체크 토글, 커스텀 추가, '병원 제공 물품' 별도 표시 토글(병원마다 다름 안내). 출력=탭별 진행률.
- **한국 맥락**: 출처 확인됨: 병원 입원 2~4일/조리원 2~3주로 짐 양이 다르므로 분리가 한국 표준 권장(마미톡·육아크루·HiPP). 산모수첩·신분증·국민행복카드는 절대 필수 최상단 고정. 병원이 제공하는 물품(산모패드·기저귀 등)은 병원마다 달라 '병원에 미리 확인' 안내 문구 표준. 출산은 예정일보다 빠를 수 있어 '예정 한 달 전 준비 + 가족에게 위치 공유' 안내가 한국 가이드 정석.
- **엣지/주의**: 산후조리원 미이용(집/친정 조리) 사용자를 위해 '조리원 가방' 탭 숨김 토글. 응급(조산) 대비 병원가방을 별도로 더 일찍 알림. 가족 공유 시 배우자가 가방 위치/체크 상태 함께 확인. 쌍둥이 시 아기 퇴원복·속싸개 수량 2배 안내.

### 출산계획서 (Birth Plan, 한국 분만 맥락)

- **무엇을**: 원하는 분만 경험을 정리해 병원·배우자와 공유하는 문서형 도구. 한국 분만 현실에 맞춘 항목으로 구성: ①분만 방식 희망(자연분만 / 제왕절개 / 자연주의출산 / 의료진 권고 따름), ②진통 완화(무통분만(경막외마취) 원함/원치않음/상황보고결정 — 한국은 무통이 사실상 주 선택지임을 반영), ③분만 중 환경(배우자 동반 입회, 음악/조명, 회음부 절개 관련 희망, 자세), ③출산 직후(피부접촉(캥거루케어) 희망, 탯줄 절단 주체, 제대혈 보관 여부), ④수유 계획(모유수유/혼합/분유 — 직후 첫 수유 시점), ⑤신생아 처치(신생아실 vs 모자동실, 예방접종/검사 동의), ⑥산후조리 계획(산후조리원/집/친정), ⑦비상시 우선순위·특이사항(알레르기, 종교적 요청). 각 항목 선택 후 1장 요약 미리보기를 만들어 배우자 공유 또는 병원 지참용으로 정리.
- **어떻게 보이나**: 건강 탭 임신 영역 '출산 준비'의 한 항목. 항목별 단계 폼(라디오 선택 + 자유 메모). 작성 후 '요약 보기' = 읽기 좋은 1장 카드(병원에 보여주기/배우자 공유용). 부부 공유 켜져 있으면 배우자도 열람·의견 메모 가능(공동 작성). 임신 3기 진입 시 작성 권유 리마인드.
- **입력**: 각 항목 선택지(라디오/체크) + 항목별 자유서술 메모. 출력=구조화된 선택값 + 1장 요약 텍스트(공유/지참용). 모두 선택 사항(빈 항목 허용).
- **한국 맥락**: 한국 통계 반영: 제왕절개 비율이 2023년 64.2%로 자연분만(35.8%)을 추월 → '제왕절개'를 결함 없는 동등 선택지로 중립 제시(죄책감 유발 카피 금지). 무통분만(경막외)이 한국에서 거의 유일한 진통완화 선택지임을 안내(호주식 수중분만 등은 국내 제한적). '자연주의출산'(조산원·차병원 등) 별도 선택지로 포함. 산후조리원은 한국 특유 문화 → 출산계획서에 조리 계획 항목 포함. 의료 면책 + '최종 결정은 담당 의료진과 상의' 필수.
- **엣지/주의**: 선택은 '희망'일 뿐 보장 아님을 명확히(의료진 판단 우선 고지). 제왕절개 확정 사용자는 자연분만 관련 진통/자세 항목 자동 비활성/접힘. 쌍둥이 등 다태아는 제왕절개 권고 비율이 높음을 중립 안내. 출산(born) 전환 후엔 읽기전용 아카이브 보존(다음 임신 시 복사 시작 옵션).

### 아기 이름 짓기 (부부 공동)

- **무엇을**: 태명(뱃속 애칭)과 출생 후 본명 후보를 부부가 함께 모으고 좁혀가는 도구. ①태명 등록(현재 임신 카드/위젯에 표시되는 애칭, 예: '튼튼이', '복덩이'), ②이름 후보 보드(부부가 각자 후보를 추가, 성별 미정/남/여 탭), ③각 후보에 부부가 '좋아요/별점' 투표 + 한줄 의견 메모, ④뜻·한자 메모 칸(작명 의미 기록), ⑤최종 후보 핀(상위 1~3개 고정). 부부 공유 기반이라 두 사람의 선호가 한 보드에 모인다. 출산 전환 시 핀 1순위를 아기 본명 후보로 prefill.
- **어떻게 보이나**: 건강 탭 임신 영역 또는 홈 임신 카드 빠른 액션 '이름'. 보드형 리스트(성별 미정/남아/여아 세그먼트). 각 카드=이름+뜻 메모+부부 각자 별점/하트. 태명은 별도 상단 입력(앱 전역 표시명에 즉시 반영). 부부 공유 ON 시 상대가 추가/투표한 항목 실시간 합류 + 작성자 표시.
- **입력**: 태명(텍스트, 앱 표시명에 사용). 이름 후보(텍스트 + 성별 분류 + 뜻/한자 메모). 부부 각자 별점/하트 투표 + 한줄 의견. 출력=후보 목록 + 합산 선호 순위 + 핀 상위 후보.
- **한국 맥락**: 한국은 태명(뱃속 애칭) 문화가 보편 → 태명을 정식 1급 기능으로(위젯·홈카드에 '튼튼이 NN주') 노출. 본명은 한자 뜻 기록이 중요(작명소·돌림자) → 뜻/한자 메모 칸 강조. 성별 미정 시기(20주 정밀초음파 전)가 길어 '성별 미정' 탭 기본. 상업 작명 서비스 연계 0(커머스 금지) — 순수 부부 의사결정 도구.
- **엣지/주의**: 성별이 초음파로 확정되면 해당 성별 탭으로 후보 정렬 권유(강제 이동 아님). 부부 투표 동점 시 순위 동률 표기. 공유 전(혼자) 사용 가능 — 배우자 합류 시 기존 후보 보존 병합. 쌍둥이는 후보 보드를 태아별(첫째/둘째)로 나눌 수 있음. 출산 후 본명 확정 시 보드 아카이브 보존.

### 홈 위젯 (D-day / 주차)

- **무엇을**: 잠금화면·홈화면 위젯으로 임신 진행을 한눈에. 크기별 3종: ①소형(앱 아이콘 옆)=D-day 큰 숫자 + 'NN주N일', ②중형=태아 크기 비유(과일/사물) + D-day + 주차 + 이번 주 한줄 마일스톤, ③잠금화면 원형(accessoryCircular)=D-day 또는 주차 진행 링. 위젯은 LMP/예정일 원본만 저장하고 날짜가 바뀌면 자동으로 '오늘 기준' 주차·D-day를 매일 새로 계산해 표시(앱 안 열어도 갱신). 태명이 등록돼 있으면 '튼튼이 D-100'처럼 애칭 표시.
- **어떻게 보이나**: iOS 위젯 갤러리에서 'BabyCare 임신' 위젯으로 추가. 소형/중형/잠금화면 3사이즈. 다크·라이트·잠금화면 틴트 모두 대응(adaptive 색). 탭하면 앱 홈의 임신 카드로 딥링크. 임신 카드 일러스트(태아)와 톤 일치.
- **입력**: 위젯이 읽는 값=예정일(또는 LMP) 원본 + 태명 + (있으면)현재 주차 마일스톤 텍스트/태아 크기 비유. 사용자 입력 없음(앱에서 등록한 임신 정보 자동 사용). 출력=매일 자동 갱신되는 D-day·NN주N일·크기비유.
- **한국 맥락**: 한국 임산부는 'NN주N일'과 'D-day'를 동시에 일상적으로 쓰므로 둘 다 노출. 태아 크기 비유는 한국에서 통용되는 비유(딸기·자두·라임 등 보유 자산 pregnancy-weeks.json 사용)로 표기. 태명 표시가 한국 정서에 큰 애착 포인트.
- **엣지/주의**: 임신 정보 미등록/임신 모드 비활성 시 위젯 데이터 비우고 '임신 정보를 등록하세요' 플레이스홀더. 출산(born) 전환 시 자동으로 임신 위젯 종료(육아 위젯으로 자연 전환 안내). 예정일 초과(D+) 시 'D-day' 아닌 'D+N 예정일 지남' 표기로 불안 최소화. 쌍둥이는 위젯에 태아 수 표기(선택). 위젯 데이터는 기존 육아 위젯 저장소와 분리(혼선 방지).

### 쌍둥이·다태아 모드

- **무엇을**: 한 임신에 태아가 2명 이상일 때 전 기능을 다태아에 맞게 동작시키는 모드. 임신 등록 시 '태아 수'(1/2/3+)를 선택하면 활성화. 영향: ①홈 임신 카드와 위젯에 '쌍둥이 NN주N일' 표기 + 태아 일러스트 2개(또는 수 배지), ②태동 기록을 태아별로 구분 가능(첫째/둘째 — 선택, 강제 아님), ③초음파 첨부·체중 등 기록에 태아 구분 라벨, ④이름 짓기 보드를 태아별로 분리, ⑤출산준비물/가방 체크리스트 수량 안내 자동 증배(카시트·침대·옷 2세트), ⑥출산계획서에 다태아는 제왕절개·조기분만 가능성이 높다는 중립 의료 안내, ⑦주차 콘텐츠에 '다태아는 진행/검진 일정이 다를 수 있음' 보조 안내. 출산 전환 시 태아 수만큼 아기 프로필을 한 번에 생성(이름·성별 각각 prefill).
- **어떻게 보이나**: 임신 등록 폼의 '태아 수' 선택 1탭으로 시작(기본 1명). 활성화되면 홈 카드·위젯·기록·체크리스트 전반에 다태아 UI가 additive로 적용(별도 탭 신설 없음 — IA 원칙 준수). 태아별 구분이 필요한 화면에서만 '첫째/둘째' 세그먼트 노출, 단태아 사용자에겐 절대 안 보임.
- **입력**: 태아 수(Int, 기본 1). 태아별 라벨(첫째/둘째/A·B 등). 태동·초음파·이름·체중 기록 시 (선택)태아 구분 태그. 출력=다태아 표기, 태아별 분리 기록 뷰, 출산 시 N개 아기 프로필 일괄 생성.
- **한국 맥락**: 한국 보유 자산 Pregnancy.fetusCount 필드 활용. 다태아는 한국에서 IVF 임신 증가로 비율 상승 → IVF 예정일 계산기와 자연 연계. 만삭사진·태교 콘텐츠도 다태아 표현 포함. 검진 일정(NT·쿼드·정밀초음파·임당)이 다태아는 빈도가 더 잦을 수 있어 '담당의 일정 따름' 안내.
- **엣지/주의**: 태아 수는 등록 후에도 변경 가능(초음파로 추가 확인되는 경우) — 변경 시 기존 기록 보존. 태아 구분은 선택사항(귀찮으면 통합 기록도 허용). 다태아 중 일부만 출산/손실되는 민감 상황은 임신 종료 로직과 연계해 데이터 삭제 없이 보존(절대 삭제 금지 원칙). 단태아→다태아 모드 전환 시 기존 단일 기록 유지. 출산 일괄 생성 시 아기별 정보 개별 수정 가능.

## F. 공유 — 부부·가족 공유 + 파트너 응원·태담 (신규 임산부 정복 목표, 카톡 친화·역할 기반)

### F-1. 초대코드로 부부·가족 잇기 (카톡 친화 초대·수락)

- **무엇을**: 임산부(주작성자)가 남편·예비조부모 등을 임신 케어에 초대하는 핵심 진입. 임신 카드 빠른액션의 '함께 보기'와 설정 > 공유에서 진입. 화면 상단에 6자리 초대코드(예: K7M2QP)가 큰 글씨로 표시되고, 아래 [카카오톡으로 초대] 버튼이 1차 CTA. 버튼을 누르면 OS 공유 시트가 떠 카톡이 최상단에 노출되며, 미리 채워진 메시지(예: 'OO이가 OO주째 자라고 있어요. 우리 아기 소식 함께 봐요 → 코드 K7M2QP, 7일 내 입력')가 함께 전달된다. 메시지 안에는 코드 + 앱 설치 안내가 같이 들어가 받는 사람이 앱이 없어도 설치 후 코드를 입력할 수 있다. 코드는 7일 후 만료되며, 만료/사용 완료 시 [새 코드 만들기]로 재발급한다. 받는 사람은 자기 앱의 설정 > 공유 > [초대코드 입력]에서 6자를 넣으면 즉시 미리보기(예: 'OO님의 임신 케어 · D-180 · 22주')가 떠 [참여하기]로 수락한다. 수락 직후 '관계' 선택(아빠/할머니/할아버지/이모·고모 등)을 한 번 묻고, 이 라벨이 멤버 목록과 응원 메시지에 함께 표시된다.
- **어떻게 보이나**: 임신 카드(홈) 빠른액션 '함께 보기' + 설정 > 공유 화면. 코드는 모노스페이스 대형 텍스트 + [복사] 아이콘. 1차 CTA=[카카오톡으로 초대](브랜드 라벨은 '공유'로 중립 표기 가능), 2차=[코드 복사]/[다른 앱으로 공유]. 수락 측은 [초대코드 입력] 6칸 코드 필드 + 수락 전 미리보기 카드. 모두 SF Symbols, ViewThatFits로 큰 글씨 접근성 대응.
- **입력**: 보유 자산 FamilyInvite(6자 코드 ABCDEFGHJKLMNPQRSTUVWXYZ23456789·혼동문자 0/1/I/O 제외·7일 만료·isUsed) + SharedBabyAccess(role viewer/editor + relationship 라벨) 그대로 사용. 기존 PregnancyShareView는 이메일로 UID를 조회하는 방식인데, 이 명세는 이를 '초대코드+카톡 공유 시트' 흐름으로 대체(이메일 입력 제거 — 한국 사용자는 이메일보다 카톡 공유가 압도적). 마미톡·핑크다이어리가 검증한 '가족과 함께 보기 + 카톡 공유'가 한국 표준임을 웹에서 확인.
- **한국 맥락**: 한국 임신앱의 가족 연결은 카카오톡 공유가 사실상 표준(마미톡=초음파 가족 공유, 핑크다이어리=남편 자동 공유). 미리 채워지는 초대 문구에 'NN주'와 'D-day'를 넣어 받는 사람이 맥락을 즉시 이해하게 한다. 코드는 입력 부담을 줄이려 6자·혼동문자 제외. 만삭 가까울수록 조부모 합류가 늘어나는 한국 가족 문화 반영(관계 라벨에 할머니/할아버지 포함).
- **엣지/주의**: ①앱 미설치자: 카톡 메시지의 설치 링크 → 설치 후 코드 입력으로 이어지게(코드 7일 유효). ②코드 만료/이미 사용됨: 입력 시 '만료된 코드예요. 초대한 분께 새 코드를 요청하세요' 안내 + 재발급 동선. ③이미 같은 임신에 참여 중인 사람이 재입력: '이미 함께 보고 있어요'로 중복 참여 방지(권한 정합성 구현은 별도 단계). ④잘못된 코드: 6자 채워야 [참여하기] 활성, 불일치 시 흔들림 피드백 + '코드를 다시 확인해 주세요'. ⑤관계 미선택: 나중에 멤버 목록에서 수정 가능, 강제하지 않음.

### F-2. 역할 기반 보기·쓰기 (누가 무엇을 보고 쓰나)

- **무엇을**: 임신 케어에 두 가지 역할이 있다. (1)주작성자=임산부 본인: 임신 등록·EDD 수정·산전검진 일정·태동/체중/증상 기록·초음파 첨부·멤버 초대/내보내기·공유 중단 등 모든 것을 보고 쓴다. (2)함께 보는 가족(공유 멤버): 임신 카드(D-day·NN주N일·크기 비유·주차 텍스트), 산전검진 일정, 초음파 사진, 주차별 콘텐츠를 '본다'. 가족 멤버에게는 두 단계가 있다 — 응원자(보기 전용): 모든 임신 콘텐츠를 보고 응원 메시지·스탬프만 남긴다(F-3). 함께기록자(쓰기 허용): 거기에 더해 산전검진 일정 체크, 초음파 사진 추가, 메모 같은 비민감 항목을 함께 입력할 수 있다. 단, 태동/체중/증상 같은 산모 몸에 관한 민감 기록은 항상 본인만 수정하고 가족은 보기만 한다. 설정 > 공유 > 멤버 목록에서 각 멤버 옆에 관계 라벨(아빠·할머니 등)과 역할 배지(보기/함께기록)가 보이고, 주작성자가 멤버를 탭해 역할을 바꾸거나 내보낼 수 있다. 멤버 본인은 자기 화면에서 '나는 보기 전용으로 참여 중'임을 명확히 안내받아, 쓸 수 없는 버튼이 왜 비활성인지 혼란이 없게 한다.
- **어떻게 보이나**: 설정 > 공유 > 멤버 목록(각 행: 관계 라벨 + 역할 배지 + 사람 아이콘). 주작성자에게만 멤버 탭 시 역할 변경/내보내기 시트 노출. 보기 전용 멤버 화면은 기록 입력 진입점을 숨기거나 '주작성자만 수정할 수 있어요' 보조 안내로 표시. 임신 카드/검진/초음파 보기는 역할 무관 공통.
- **입력**: 보유 자산 SharedBabyAccess.role(viewer/editor)을 사용자 언어 '보기/함께기록'으로 노출, relationship 라벨 병기. 민감 기록(태동/체중/증상)은 항상 본인 전용이라는 규칙은 임신 데이터 민감성(safety.md: 임신 데이터 Analytics 금지와 같은 보호 기조)과 일치. 권한 강제·데이터 경로(dataUserId 라우팅)는 구현 단계이므로 이 명세는 '역할별로 무엇이 보이고 가능한가'의 UX만 규정.
- **한국 맥락**: 한국 가족은 조부모까지 '함께 보기'를 원하지만 임신 기록을 직접 건드리길 원하지는 않는 경우가 많음 → 기본 역할은 '보기(응원자)'로 두고, 남편처럼 적극 참여자만 '함께기록'으로 승격하는 단계 구조가 자연스럽다. 산모 몸 관련 민감 기록을 본인 전용으로 격리해 가족 공유가 부담이 되지 않게 한다.
- **엣지/주의**: ①보기 전용 멤버가 잠긴 기록 버튼 발견: 비활성 버튼 대신 '주작성자만 기록할 수 있어요' 안내로 자괴감/혼란 방지. ②주작성자가 자기 역할을 낮출 수 없음(항상 전체 권한 유지 — 케어 주인 보호). ③멤버를 내보낸 뒤: 그 사람 화면에서는 더 이상 임신 카드가 보이지 않고 '공유가 종료되었어요' 안내. ④함께기록자가 민감 기록 화면 진입 시: 보기는 되지만 편집은 잠김(태동/체중/증상). ⑤멤버 0명 상태: 멤버 목록 빈 화면에 '아직 함께 보는 가족이 없어요 · 카카오톡으로 초대해 보세요' 빈 상태 안내 + F-1 진입 버튼.

### F-3. 파트너 응원 메시지·태담 스탬프

- **무엇을**: 함께 보는 가족(특히 남편)이 임신 카드와 주차 소식 위에 짧은 응원을 남기는 정서적 핵심 기능. 임신 카드 또는 주차별 소식 화면 하단에 [응원 남기기] 버튼이 있고, 누르면 (1)스탬프 빠른 선택: 손하트·토닥토닥·뽀뽀·화이팅·천사·축하 같은 임신/태담 정서의 6~8종 스탬프를 한 번에 톡 보내거나, (2)짧은 한 줄 메시지(예: '오늘도 잘 견뎌줘서 고마워', '아빠야~ 무럭무럭 자라렴')를 직접 입력한다. 남긴 응원은 시간순 타임라인으로 임신 카드에 쌓이고, 각 응원에는 보낸 사람의 관계 라벨(아빠·할머니)과 시간이 붙어 '누가 응원했는지'가 한눈에 보인다. 임산부는 받은 응원에 ♥로 반응할 수 있다. 태담 문화에 맞춰, 아기에게 말 거는 톤의 메시지(태담 한마디)를 권장 문구로 몇 개 제시해 첫 입력 장벽을 낮춘다. 응원이 새로 도착하면 임신 카드에 작은 새 알림 점이 표시되어 함께 보는 즐거움을 만든다.
- **어떻게 보이나**: 임신 카드(홈) 하단 응원 타임라인 + [응원 남기기] CTA. 스탬프는 가로 스크롤 칩/그리드(SF Symbols 기반 일러스트형, 손하트/하트/별 등) + 선택 시 카드에 말풍선으로 표시. 한 줄 메시지는 짧은 입력 시트 + 권장 태담 문구 칩. 응원 도착 시 카드에 빨간 점 인디케이터.
- **입력**: 신규 정서 기능(보유 자산에 없음 — 부부공유 위에 추가). 스탬프 세트·권장 태담 문구는 신규 콘텐츠 자산. 보낸 사람 관계 라벨은 F-1/F-2의 relationship 재사용. 상업요소 0(유료 스탬프·광고 스탬프 없음).
- **한국 맥락**: 한국의 태담 문화(아기에게 말 걸기)를 정면으로 반영 — 단순 '좋아요'가 아니라 '아빠가 아기에게 한마디' 형태의 태담 한마디를 권장 문구로 제시. 부부가 함께 임신을 겪는다는 정서적 연결이 신규 임산부에게 가장 강한 락인. 마미톡류가 정보 공유에 그치는 데 반해, 감정 공유(응원·태담)로 차별화.
- **엣지/주의**: ①보기 전용·함께기록 역할 모두 응원은 가능(응원은 민감 기록이 아니므로 모든 가족에게 열어 참여 유도). ②임산부 본인도 자기 카드에 태담을 남길 수 있음(혼자여도 빈 타임라인이 어색하지 않게). ③응원 길이 제한(한 줄, 예 60자)으로 잡담화 방지 + 가벼운 톤 유지. ④부적절/상처 줄 수 있는 메시지: 보낸 응원은 임산부가 숨기거나 삭제할 수 있어 본인이 타임라인을 통제. ⑤멤버가 0명일 때: '함께 볼 가족을 초대하면 응원을 받을 수 있어요'로 F-1 유도, 본인 태담은 여전히 가능. ⑥임신 종료(출산/유산 등) 후: 과거 응원 타임라인은 아카이브로 보존(데이터 삭제 절대 금지 기조와 일치), 새 응원 입력만 닫힘. ⑦민감성: 응원 메시지·스탬프 내용은 임신 데이터 보호 원칙에 따라 분석 페이로드에 포함하지 않는다(safety.md 임신 데이터 Analytics 금지).

## G. 영양·식단 (의료감수)

### 음식 안전 조회 — '이거 먹어도 돼요?' 검색

- **무엇을**: 임신부가 특정 음식이나 음료를 '먹어도 되는지' 그 자리에서 검색해 신호등(안전/주의/제한/피하기 4단계) 판정과 그 이유를 의료감수 카피로 확인하는 기능. 입덧·식욕 변화·외식 상황에서 가장 자주 떠오르는 불안('회 한 점 먹었는데 괜찮나', '커피 마셔도 되나', '이 약 같이 먹어도 되나 말고 음식 위주')을 즉답으로 해소한다. 한국 음식·외식 메뉴를 1급 시민으로 포함한다.

[판정 4단계와 의미]
- 초록(안전): 일반적인 양으로 먹어도 무방. 예) 잘 익힌 소고기, 익힌 채소, 두부, 김, 사과.
- 노랑(주의/양 제한): 먹어도 되지만 양·빈도·조리법 조건이 붙음. 예) 커피(하루 카페인 300mg 이하, 약 2잔), 참치회/다랑어 등 대형어류(메틸수은 — 주당 제한), 참치캔(주당 400g까지), 일반 생선(주당 약 400g).
- 주황(강한 제한/조건부): 특정 조리·살균 조건에서만. 예) 치즈(살균유로 만든 단단한 치즈는 OK / 브리·카망베르 등 비살균 연성치즈는 피하기), 생햄·살라미 등 가공·비가열 육류.
- 빨강(피하기): 임신 중 권장하지 않음. 예) 술(전 기간 금주 — 안전한 양 없음), 회·육회·생굴 등 날것, 덜 익힌 고기/달걀, 자라·한약재 일부 등.

[검색 경험]
- 한 줄 검색창에 음식 이름을 입력하면 입력 즉시 후보 목록이 좁혀짐(예: '회' → 광어회/연어회/육회/회덮밥, '커피' → 아메리카노/라떼/디카페인). 한글 자모 부분 입력·동의어(참치=다랑어, 회=사시미)도 인식.
- 결과 카드 = ①음식명 + 큰 신호등 색/아이콘 ②한 줄 결론('하루 1~2잔까지 괜찮아요') ③이유(왜 그런지 — 리스테리아/메틸수은/카페인/톡소플라스마 등 위험 요인을 쉬운 말로) ④실천 팁('카페인은 커피 외 녹차·콜라·초콜릿에도 들어있어 합산해 보세요') ⑤의료감수 면책 문구.
- 검색어가 목록에 없으면 '아직 등록되지 않은 음식이에요' + 가장 가까운 카테고리 일반 원칙(예: 날것은 피하고, 잘 익혀 드세요)을 안내하고, '주치의에게 확인하세요' 폴백 카피. 추측 단정 금지.

[탐색(검색 외) 경험]
- 검색창 아래 카테고리 칩: 음료(카페인)/생선·해산물/육류·가공육/유제품·치즈/회·날것/한식·외식/과일·채소/한약·건강식품. 칩을 누르면 해당 카테고리 음식이 신호등별로 정렬되어 브라우징 가능.
- '많이 찾는 질문' 모듈: 회·커피·참치캔·치즈·생맥주·홍어/육회 등 한국 임신부가 실제로 가장 많이 검색하는 항목을 상단 고정.
- 최근 본 음식 / 즐겨찾기(별표)로 자주 확인하는 음식을 빠르게 재조회.
- **어떻게 보이나**: 건강 탭 내 '음식 안전' 진입점(섹션 카드) + 홈 임신 카드의 빠른 액션 중 '먹어도 돼요?' 1개. 진입 시 상단 고정 검색창 + 카테고리 칩 + '많이 찾는 질문' + 최근/즐겨찾기. 검색 결과·카테고리 결과는 신호등 색을 좌측 굵은 컬러바/아이콘으로 표현해 스캔 가능하게. 결과 카드는 push 상세(결론→이유→팁→면책 순서). 신호등 색은 색각 보조를 위해 색+아이콘+텍스트 라벨 3중 표기(초록 체크/노랑 느낌표/빨강 금지). 모든 의료 결과 화면 하단 면책 배너 고정.
- **입력**: 사용자 입력 = 음식/음료 이름 텍스트 검색, 카테고리 칩 탭, 즐겨찾기 토글. 시스템 입력 = 의료감수 완료된 음식 안전 콘텐츠(음식명·동의어·신호등 등급·이유·양 제한·팁·출처/감수일). 현재 임신 주차(트라이메스터)를 알면 시기 특이 주의를 가산(예: 초기엔 톡소플라스마·엽산 강조). 콘텐츠는 앱 내장 + 갱신 가능한 데이터로 운영(외부 광고/커머스 링크 없음).
- **한국 맥락**: 한국 식약처/질병관리청 기준 반영 필수 — 카페인 하루 300mg 이하(약 2잔), 일반 생선 주당 약 400g, 참치캔 주당 400g까지, 대형어류(다랑어·심해어) 메틸수은 주의로 섭취량 제한, 술은 전 기간 금주(안전량 없음 — 태아알코올증후군), 회·육회·날것은 리스테리아/톡소플라스마/비브리오 위험으로 피하기, 비살균 연성치즈·생햄 제한. 한국 특이 항목 필수 포함: 회/사시미·초밥, 홍어·육회, 한약재·녹용·자라, 인삼·홍삼, 멸치·정어리 같은 소형어(오메가3 안전 대안), 김·미역(요오드 — 과다 주의), 생맥주·막걸리. 동의어 한국어(참치=다랑어, 회=사시미, 날것). 모든 텍스트 한국어, NN주N일 시기 맥락.
- **엣지/주의**: ①등록 안 된 음식 → 단정 금지, 카테고리 일반 원칙 + '주치의 확인' 폴백. ②'노랑/주황'을 '안전'으로 오인하지 않도록 결론 문구에 조건을 반드시 병기(예: '디카페인은 OK, 일반 커피는 하루 2잔까지'). ③카페인은 합산 개념(커피+녹차+초콜릿) 안내 — 단일 식품만 보고 안심하는 오류 방지. ④알레르기·기저질환(임신성 당뇨 등)은 개인차 큼 → 일반 가이드일 뿐 개별 의학 판단 아님 면책. ⑤색각 이상 사용자 위해 색 단독 의존 금지(아이콘+라벨). ⑥오프라인에서도 핵심 음식은 조회 가능하도록 내장 우선. ⑦검색 0건/오타 시 가까운 후보 추천. ⑧'먹어버린 후' 검색하는 불안 사용자 톤 배려 — 빨강이라도 비난조 금지, 차분히 '다음부터는' + 필요 시 '걱정되면 의료진 상담' 안내.

### 트라이메스터별 영양 가이드 + 영양제(엽산·철분 등) 코치

- **무엇을**: 임신 시기(초기/중기/후기 = 트라이메스터)에 맞춰 '지금 무엇을 더 챙겨야 하는지'를 영양소 단위로 안내하고, 한국 권장 영양제(엽산·철분·칼슘·비타민D·요오드·오메가3/DHA)를 언제·얼마나 먹어야 하는지 시기별로 코치하는 기능. 현재 임신 주차에 따라 카드 내용이 자동으로 바뀌어, 매번 검색하지 않아도 '이번 시기 핵심'이 떠먹여진다.

[시기별 핵심(한국 기준 반영)]
- 임신 준비~초기(~13주): 엽산 강조 — 하루 600μg 권장(신경관 결손 예방), 임신 3개월 전부터 13주까지 특히 중요. 입덧으로 못 먹을 때 소량씩 자주, 수분 보충 팁.
- 중기(약 14~27주): 철분 강조 — 하루 24mg 권장(혈액량 급증·빈혈 예방, 보통 임신 5개월경부터 보충 권장, 빈혈 진단 시 증량은 의료진 처방). DHA(두뇌·망막 발달), 단백질·칼슘 병행.
- 후기(28주~출산): 칼슘(하루 약 280mg 추가) — 태아 골격·치아, 산모 뼈 보호. 철분 지속, 변비·부종 관리 식이, 출산 대비 에너지.

[영양제 코치]
- 영양소 카드 각각: 무엇에 좋은지(쉬운 말)·하루 권장량(한국 기준 수치)·언제부터/언제까지·흡수 팁(예: 철분은 비타민C와 함께, 커피·칼슘과 시차 / 칼슘과 철분은 동시 복용 시 흡수 경쟁)·과다 주의(요오드·비타민A 과다 위험).
- '내 영양제 챙기기' 체크: 오늘 엽산/철분 등 복용 여부를 탭 한 번으로 기록(가벼운 셀프 체크 — 의무 아님), 캘린더/건강 탭에서 복용 흐름을 돌아볼 수 있음. 알림(선택)으로 복용 리마인드.
- 식품으로 채우기: 영양소별 '한국 음식으로 보충하는 법'(엽산=시금치·아보카도·콩, 철분=소고기·시금치·두부, 칼슘=멸치·두부·유제품, 요오드·DHA=소형어·미역 — 단 과다 주의). 음식 안전 조회와 교차 연결.

[전문가 콘텐츠]
- 보유 자산 pregnancy-weeks.json(37주 텍스트)의 주차별 tip과 결합해, 그 주차의 발달 이벤트와 영양 포인트를 한 흐름으로 제시(예: 5주 '심장 형성 시작 — 엽산 챙기기', 20주 '망막 발달 — DHA').
- **어떻게 보이나**: 건강 탭 '영양 가이드' 섹션 + 홈 임신 카드에서 현재 시기 영양 핵심 1줄 미리보기('이번 주 핵심: 철분 챙기기'). 진입 시 ①현재 트라이메스터 헤더(NN주N일·D-day 연동) ②이번 시기 핵심 영양소 카드 스택(중요도 순) ③영양제 코치(권장량·시기·흡수 팁) ④'오늘 영양제 체크' 가벼운 토글 ⑤식품으로 보충하기 리스트. 트라이메스터 전환 시 카드 구성 자동 변경. 복용 체크는 기록 탭/캘린더에 가볍게 표시(육아 기록과 톤 일치). 의료 수치 표기 화면 하단 면책 배너 + 감수 표기('산부인과 전문의 감수').
- **입력**: 사용자 입력 = 영양제 복용 체크 토글, 알림 on/off, 카드 탭(상세). 시스템 입력 = 현재 임신 주차/트라이메스터(LMP·EDD 기반 자동 계산, 기존 NN주N일 로직 재사용), 의료감수 영양 콘텐츠(영양소별 권장량·시기·팁·식품 매핑), pregnancy-weeks.json 주차 텍스트. 개인 맞춤은 일반 가이드 범위 내에서 시기 자동 분기까지만(개별 처방 아님).
- **한국 맥락**: 한국 권장 수치 그대로 — 엽산 600μg/일(임신 3개월 전~13주 강조), 철분 24mg/일(중기·약 5개월부터, 빈혈 시 증량은 의료진), 칼슘 약 280mg/일 추가(후기), 비타민D 약 10μg/일, 요오드·DHA 권장하되 과다 주의. 한국 음식 기반 식품 보충(시금치·두부·멸치·미역·소고기·소형어). 산모수첩/국민행복카드·산전검진(11~13주 NT, 16주 쿼드, 20주 정밀초음파, 24~28주 임당) 일정과 시기적으로 자연 연결(예: 임당 검사 시기엔 당·탄수 식이 안내). 모든 콘텐츠 한국어, 시기 표기는 NN주N일+D-day. 태담·만삭사진 등 한국 임신 문화 톤 유지하되 영양은 의료 사실 위주.
- **엣지/주의**: ①권장량 수치는 일반 기준 — 빈혈·임신성 당뇨·다태아 등은 개별 처방이 우선이며 '의료진 지시가 항상 우선' 면책 필수. 앱이 증량/감량을 지시하지 않음. ②영양제 복용 체크는 '의무'가 아님을 톤으로 명확히(놓쳐도 죄책감 주지 않기 — 데이터 삭제 금지 룰과 같은 사용자 존중 정서). ③과다 위험 영양소(요오드·비타민A·미역 과다)는 '많을수록 좋음' 오인 방지 경고. ④흡수 상충(철분↔칼슘/커피) 안내로 잘못된 동시 복용 방지. ⑤트라이메스터 경계(13/14주, 27/28주)에서 카드가 자연스럽게 넘어가되 갑작스러운 '끝났어요' 단절감 없이 전환. ⑥임신 주차 정보가 없거나 출산 전환(born/유산 등) 상태면 영양 코치를 노출하지 않음(민감 상태 배려 — baby>pregnancy gating 정신과 일치). ⑦수치/시기 콘텐츠는 감수일·출처를 운영적으로 추적해 기준 개정 시 갱신 가능하게(예: 한국인 영양소 섭취기준 개정).

## H. 전환 (임신 → 출산 → 육아 무손실 연결)

### 출산 CTA 게이트 (36주부터 등장하는 '아기 만났어요' 버튼)

- **무엇을**: 임신 36주 0일에 도달하면 홈의 임신 케어 카드 안에 '아기 만났어요 / 출산했어요' 행동 버튼이 처음 등장한다. 36주 이전에는 절대 노출되지 않는다(아직 출산 시점이 아닌 산모에게 출산 종료 유도를 보이지 않기 위함 = 정서적 안전선). 36주 이후로는 카드의 빠른 액션 줄(태동·체중 등)과 나란히, 또는 카드 하단에 부드러운 강조 톤(브랜드 핑크 #FF9FB5 라인/은은한 채움)으로 상주한다. 출산 예정일(D-day)이 0에 가까워질수록 자연스럽게 더 눈에 띄도록 카피가 변한다: 36~39주 '아기를 만나면 알려주세요', D-7 이내 '곧 만날 시간이에요 · 출산하면 눌러주세요', D-day 경과(예정일 초과) 시 '출산하셨나요? 아기 프로필로 옮겨드릴게요'. 사용자가 아직 출산 전이면 버튼을 누를 일이 없고, 버튼은 강요하지 않는다.
- **어떻게 보이나**: 홈 탭 임신 케어 프로필 카드 내부. 별도 화면/탭 없음. 36주 게이트 통과 시 카드 안에 인라인 등장. 톤은 축하 예고(설렘) — 경고/필수 느낌 금지.
- **입력**: 탭하면 '출산 정보 입력' 화면(별도 기능)이 모달로 열린다. 탭 외 입력 없음.
- **한국 맥락**: 한국 산전 기준상 37주부터 만삭(정상 만삭아)이지만, 35~36주 조기 출산도 흔해 게이트를 36주 0일로 둬 조산 케이스도 누락 없이 전환 가능하게 한다. D-day/NN주N일 체계와 일관(카드 상단 D-day 라벨 그대로 재사용). '출산 예정일'은 한국 산모에게 가장 익숙한 기준점이라 카피가 이를 중심으로 변한다.
- **엣지/주의**: 쌍둥이 등 다태아도 동일 게이트(36주). 예정일이 지나도(D+) 버튼은 사라지지 않고 카피만 '출산하셨나요'로 바뀌어 계속 상주(예정일 초과 출산 대응). 36주 미만에서 실제 조산한 사용자를 위한 우회: 설정 > 임신 관리 > 출산 등록 심층 경로는 주차 무관하게 항상 접근 가능(게이트는 홈 카드의 자동 노출 시점일 뿐, 등록 자체를 막지 않음).

### 출산 CTA Dismiss (다음에 / 안내 접기)

- **무엇을**: 36주 후 등장한 출산 CTA가 부담스러운 사용자를 위해 '다음에' 또는 카드 우상단 X로 이번 노출을 접을 수 있다. Dismiss하면 출산 버튼이 며칠간(예: 3~5일) 카드에서 작아지거나 숨겨지고, 이후 다시 부드럽게 재등장한다. 단, 예정일을 지난 시점(D+)에는 dismiss하더라도 '출산하셨나요?' 한 줄 안내는 유지(전환 누락 = BabyCare 최대 손실이므로 완전 소거는 안 함). Dismiss는 데이터를 전혀 바꾸지 않으며 임신 추적(주차·태동·체중·체크리스트)은 그대로 계속된다.
- **어떻게 보이나**: 출산 CTA 옆/위의 작은 '다음에' 텍스트 버튼 또는 카드 모서리 닫기 아이콘. 누르면 즉시 접힘 애니메이션. 토스트나 추가 확인 없음(가벼운 인터랙션).
- **입력**: 탭 1회. 별도 입력 없음. 되돌리려면 며칠 뒤 자동 재등장 또는 설정의 출산 등록 경로로 언제든 진입.
- **한국 맥락**: 출산은 예정일과 실제일이 며칠씩 다른 게 정상(예정일은 통계적 중앙값)이라, 한국 산모도 예정일 전후 1~2주 분포가 흔하다. 그래서 dismiss 후 재등장 간격을 짧게 둬 실제 출산 타이밍을 놓치지 않게 한다.
- **엣지/주의**: dismiss 상태에서도 임신 카드 자체(D-day·주차)는 정상 표시. 앱 재설치/기기 변경 시 dismiss 기록이 초기화돼 CTA가 다시 보일 수 있음(허용 — 누락보다 안전). 다태아·종료(유산/사산) 경로와는 무관(종료는 설정 심층 경로 전용).

### 출산 정보 입력 → 아기 프로필 자동 생성 (전환의 핵심)

- **무엇을**: 출산 CTA를 누르면 '출산 정보 입력' 화면이 열린다. 입력 항목: ①아기 이름(태명이 있으면 자동 채움, 수정 가능) ②성별(초음파 확인 성별이 있으면 자동 선택, 수정 가능) ③실제 출생일(기본 오늘, 미래일 선택 불가) ④출생 시각(선택) ⑤출생 체중(선택, kg) ⑥출생 키/신장(선택, cm) ⑦머리둘레(선택, cm) ⑧아기 수(임신 태아 수가 2명 이상이면 자동으로 '쌍둥이 모드'가 켜져 아기별로 이름·성별·체중·키를 각각 입력). '아기 만났어요' 확정 시 입력한 정보로 정식 아기 프로필이 자동 생성되고, 출생 체중·키·머리둘레가 있으면 아기의 첫 성장 기록(생후 0일)으로 같이 저장돼 성장 차트의 출발점이 된다. 완료되면 축하 화면(아기 이름 + 체크 애니메이션)이 잠깐 뜬 뒤 홈으로 돌아가면, 임신 카드 자리에 새 아기 케어 카드가 들어선다.
- **어떻게 보이나**: 홈 임신 카드의 CTA → 모달 시트(중간/전체 높이). 기존 '출산 완료 등록' 시트를 본격 추적기 수준으로 확장(현재는 이름/성별/출생일만 → 시각·체중·키·머리둘레·아기 수 추가). 확정 전 '아기 프로필로 전환할까요?' 확인 1회. 성공 시 축하 화면 약 2초 후 자동 닫힘.
- **입력**: 이름(텍스트, 필수), 성별(분절 선택), 출생일(날짜, 오늘 기본·미래 불가, 한국어 로케일), 출생시각(시간, 선택), 체중·키·머리둘레(숫자, 선택), 아기 수(태아 수≥2면 자동 다중 입력 폼). 모든 의학 수치는 선택 — 비워도 전환 가능.
- **한국 맥락**: 한국 신생아 출생 정보의 표준 계측 3종 = 체중·신장(키)·머리둘레(두위). 한국 평균 출생 체중은 남아 약 3.3kg·여아 약 3.2kg, 신장 약 49~50cm, 머리둘레 약 33~36cm이며 정상 범위는 체중 2.5~4.0kg·신장 45~55cm. 입력 필드의 플레이스홀더/예시값에 이 범위를 힌트로 제시해 산모가 산모수첩·출생증명 수치를 그대로 옮기기 쉽게 한다. 의학적 정상/비정상 판정 텍스트는 표시하지 않음(면책 원칙).
- **엣지/주의**: 쌍둥이/삼둥이: 임신의 태아 수가 그대로 넘어와 아기 수가 미리 채워지고, 첫째/둘째 등 아기별 개별 입력. 한 명만 출생(다태아 중 일부 손실)한 민감 케이스는 아기 수를 줄여 입력 가능. 체중/키 미입력 시 첫 성장 기록은 생성하지 않고 프로필만 생성(나중에 성장 탭에서 추가). 출생일을 과거로 지정하면 그 날부터 생후 일수가 계산됨. 입력 중 앱이 꺼져도 임신 데이터는 그대로 남아 다시 시도 가능(전환 도중 실패 시 복구 안내 모달이 자동으로 뜸).

### 임신 기록의 무손실 보존 (아기 타임라인 안 임신 챕터)

- **무엇을**: 출산 전환이 끝나면 임신 중 쌓은 모든 기록 — 주차별 성장 텍스트, 태동(킥) 세션, 산모 체중 추이, 산전검진(NT·쿼드·정밀초음파·임당 등) 일정과 결과, 초음파 사진 첨부, 증상 일지, 체크리스트 — 이 사라지지 않고 새로 생긴 아기의 타임라인 맨 앞에 '임신 기록(또는 태명 시절)' 챕터로 그대로 이어 붙는다. 사용자는 새 아기 프로필의 기록/캘린더에서 '출생 이전' 구간으로 거슬러 올라가 임신 시절 태동·초음파·체중을 다시 볼 수 있다. 즉 임신→육아가 하나의 끊김 없는 한 아이의 이야기로 연결된다. 임신 카드는 사라지고 아기 카드로 대체되지만, 그 안의 과거 임신 데이터는 100% 열람 가능.
- **어떻게 보이나**: 새 아기 프로필의 캘린더/타임라인 상단 또는 가장 과거 구간에 '임신 기록' 섹션으로 표시. 별도 임신 탭은 없음(IA 원칙). 설정 > 이전 임신에서도 동일 아카이브 열람 가능.
- **입력**: 사용자 입력 없음 — 전환 시 자동 연결. 열람은 타임라인 스크롤/탭.
- **한국 맥락**: 한국 태담(胎談) 문화와 태명 정서상, 출산 후에도 태명 시절 기록(태동·초음파)을 추억으로 다시 보고 싶어하는 수요가 크다. '태명 → 본명' 연결을 명시(예: '○○(태명) 시절의 기록')해 정서적 연속성을 강조. 만삭사진·초음파 사진도 같은 타임라인에서 출생 직후 사진과 자연스럽게 이어진다.
- **엣지/주의**: 다태아 전환 시 임신 기록(공통: 산모 체중·산전검진은 임신 단위, 태동은 가능하면 아기별)은 각 아기 타임라인에 어떻게 귀속할지 — 공통 임신 데이터는 모든 아기 타임라인에서 동일 임신 챕터로 공유 열람. 부부 공유 중이던 임신은 출산 후에도 공유 보호자가 아기 임신 기록을 함께 볼 수 있게 보존. 임신 종료(유산/사산)는 이 축하 전환이 아니라 설정의 별도 위로 경로로 처리되며 기록은 '이전 임신'에 보존.

### 전환 후 부부/보호자 공유 및 위젯 자동 인계

- **무엇을**: 임신 기간 동안 파트너와 공유 중이었다면, 출산 전환 후 새 아기 프로필도 같은 보호자에게 자동으로 공유 연결돼 배우자가 별도 재설정 없이 곧바로 아기 기록을 함께 볼 수 있다. 또한 잠금화면/홈화면의 임신 D-day 위젯은 출산과 동시에 임신 카운트다운을 멈추고, 새 아기 기준(생후 일수·다음 수유/낮잠 등 육아 위젯)으로 전환되거나 정리된다. 사용자가 '임신앱'에서 '육아앱'으로 갈아타는 느낌 없이, 같은 앱·같은 가족·같은 위젯이 매끄럽게 모드만 바꾼다.
- **어떻게 보이나**: 공유는 무화면 자동 인계(설정 > 가족 공유에서 확인 가능). 위젯은 홈/잠금화면에서 자동 갱신(임신 D-day 위젯 → 육아 위젯). 전환 직후 홈은 임신 카드 대신 아기 케어 카드.
- **입력**: 사용자 입력 없음 — 전환 시 자동. 필요 시 설정에서 공유 보호자 추가/수정.
- **한국 맥락**: 한국에서 출산 전후 부부 공동 기록 수요가 높고(아빠도 태동·검진 일정 공유), 출산 직후 정신없는 시기에 재설정을 강요하면 이탈한다. 자동 인계로 '출산했는데 앱이 또 처음부터'라는 마찰을 제거 — 신규 임산부 정복 목표의 리텐션 핵심.
- **엣지/주의**: 위젯은 임신 데이터 원본이 비워지면 임신 D-day 위젯이 자동으로 정리(빈 상태/제거)되어 출산 후에도 'NN주'가 박제되는 버그 없음. 공유 보호자가 임신은 보던 사람과 출산 후 다를 경우 설정에서 조정. 전환이 중간에 끊긴(앱 종료) 경우, 다음 실행 시 '출산 전환을 마저 진행할까요?' 복구 모달이 떠서 공유/위젯 인계가 누락 없이 완결되도록 보장.

## 영역 I. 손실 경로 (유산/사산/임신중지) — 조용한 모드(Quiet Mode)

### 소프트 단일 진입 (조용한 모드 진입)

- **무엇을**: 임신 추적을 더 이상 이어갈 수 없게 된 사용자가, 임상 용어나 결과 분류를 강요받지 않고 단 하나의 부드러운 입구로 들어오는 경로. 진입점 카피는 결과 유형(유산/사산/중지)을 묻지 않고 '임신을 더 이상 이어가지 못하게 되었어요' 같은 중립적·비귀책적 문구 단 하나로 제시한다. 사용자가 이 입구를 누르면 곧장 데이터가 사라지거나 모드가 전환되지 않고, 먼저 '잠시 머무는' 안내 화면(무엇이 어떻게 바뀌는지, 무엇은 그대로 보존되는지)을 보여준 뒤 사용자가 다시 한 번 명시적으로 확정해야 조용한 모드로 들어간다. 입구 자체는 시각적으로 절대 강조하지 않는다(빨강·경고 아이콘·destructive 빨강 버튼 금지, 회색조 텍스트 링크 수준). 결과 유형(유산<20주 / 사산≥20주 / 임신중지)은 카피 정확성과 행정 안내를 위해 내부적으로만 분기하며, 사용자에게는 선택적으로 '원하시면 유형을 남겨두실 수 있어요(건너뛰어도 괜찮아요)' 형태로만 제시한다. 강제 드롭다운 금지.
- **어떻게 보이나**: 두 곳에 조용히 노출. (1) 설정 > 임신 관리 > 맨 아래, 다른 메뉴와 구분된 옅은 회색 텍스트 링크 '임신을 이어가지 못하게 되었어요'. (2) 홈 임신 카드의 빠른 액션 더보기(...) 안 가장 마지막 항목으로만(카드 표면 1급 버튼 금지). 누르면 풀스크린 시트(소프트 안내 → 확정)로 전개. 시트 1단계: 위로 한 문장 + '바뀌는 것/그대로인 것' 2열 요약 + '천천히 하세요, 지금 결정하지 않아도 됩니다' + [계속] / [나중에]. 2단계(선택): 유형 옵션(유산·사산·중지·'지금은 고르지 않기' default) + 날짜(기본 오늘, 수정 가능) + [조용한 모드로 전환]. 어디서든 시트 바깥 탭/스와이프로 빠져나와도 아무 일도 일어나지 않음.
- **입력**: 사용자 탭(진입), 선택적 결과 유형 1개(기본 미선택), 선택적 손실 날짜(기본 오늘). 그 외 입력 강제 0. 기존 임신 데이터(주수·태명·태동·체중·초음파·메모)는 입력 없이 자동 보존.
- **한국 맥락**: 결과 유형의 임상 경계는 한국 기준에 맞춤: 임신 20주 미만 자연 종료=유산, 20주 이상=사산이 통용되나, 행정·휴가 안내는 16/22/28주 구간(근로기준법 유산·사산휴가)을 따른다. 사용자에게는 이 숫자를 노출하지 않고 내부 분기에만 사용. '낙태'라는 단어는 카피에서 전면 금지(임신중지로 통일, 2021년 형법 낙태죄 효력 상실 이후 중립 용어). 진입 카피는 '실패' '잘못' '안타깝게도 실패하셨' 등 귀책·실패 함의 단어 절대 금지.
- **엣지/주의**: (1) 파트너(공유 받은 사람)에게는 이 진입점 자체를 노출하지 않음 — 비가역 결정은 소유자 한정. (2) 다태아 중 일부만 손실된 경우는 v1 범위에서 단일 결과로만 처리(부분 손실 세분화는 추후), 카피는 '한 아이를 잃으셨다면…' 같은 단정 금지. (3) 이미 출산(born) 봉인된 임신에는 진입점 미노출. (4) 진입 시트 1단계에서 이탈해도 임신 상태는 ongoing 그대로 — 부분 진입으로 데이터가 어중간해지지 않음. (5) 임신 카드와 baby 카드가 공존(.both)할 때 진입 시 baby 데이터·UI는 전혀 영향받지 않음(임신 카드만 조용히 전환).

### 주수 freeze (D-day·NN주N일 카운팅 정지)

- **무엇을**: 조용한 모드로 전환되는 즉시, 매일 자동 증가하던 'NN주N일'과 'D-day'가 손실 시점에 동결된다. 더 이상 '오늘 16주 3일' 같은 진행 카운팅을 보여주지 않고, '주차가 계속 늘어나며 상실을 매일 상기시키는' 가장 큰 정서적 가해 요소를 제거한다. 주차 콘텐츠(이번 주 태아 크기 비유·발달 마일스톤·주차별 팁), 산전검진 D-day, 태교/체중/검진 알림 등 '앞으로'를 가정하는 모든 진행형 요소가 자동으로 소멸한다. freeze된 시점의 주차는 사용자가 원하면 추모 기록 안에서만 조용히 확인 가능(예: '15주 2일에 함께했어요')하되, 홈·위젯·알림 어디에서도 능동적으로 표시하지 않는다.
- **어떻게 보이나**: 전환 즉시 홈의 임신 카드(태아 일러스트+D-day+NN주N일+크기비유+빠른액션) 전체가 사라지고, 자리에 아무 진행 정보도 없는 매우 차분한 '함께한 시간' 추모 카드(선택 시) 또는 카드 없음 상태로 대체. 잠금화면·홈 위젯의 D-day 위젯은 다음 갱신 주기에 즉시 비워지거나 제거됨(숫자 잔상 금지). 캘린더의 예정일·검진 일정 마커도 회색 처리 후 비활성. 어떤 화면에서도 '며칠 남음' '몇 주차' 능동 표시 0.
- **입력**: 사용자 입력 없음(전환 시 자동). 손실 날짜를 기준으로 freeze 시점 계산. 사용자가 손실 날짜를 수정하면 freeze 표기도 함께 조정.
- **한국 맥락**: 한국식 'NN주N일 + D-day' 표기가 임신 추적의 핵심 정체성이므로, 이것이 멈추는 것 자체가 가장 강한 정서적 신호다. freeze 후 주차를 추모 맥락에서 보여줄 때도 '몇 주 만에 떠났다'는 식의 부정적 프레이밍 금지, '우리가 함께한 NN주N일' 같은 동행 프레이밍 사용. 한국 산모수첩 문화상 주차는 의미가 큰 숫자이므로 완전 삭제가 아니라 '조용히 보존, 능동 비노출' 원칙.
- **엣지/주의**: (1) 위젯은 별도 데이터스토어로 갱신되므로 전환 즉시가 아니라 다음 타임라인 갱신 시 비워질 수 있음 → 전환 직후 위젯 강제 갱신 트리거 필요(숫자 잔상이 가장 큰 2차 가해). (2) 손실 날짜를 미래로 잘못 입력해도 음수 D-day가 표시되지 않도록 클램프. (3) 전환 도중 앱이 종료되어 freeze가 미완(pending)이면, 재실행 시 진행 카운팅이 절대 부활하지 않도록 — 조용한 모드는 '진행 부활'을 우선 차단하고 미완 복구는 조용히 처리. (4) 검진 알림이 시스템에 이미 예약된 경우 freeze 시점에 즉시 전부 취소(하드취소 기능과 연동).

### 알림 하드취소 (예약된 모든 임신 알림 즉시·완전 무력화)

- **무엇을**: 조용한 모드 전환 즉시, 이미 예약·발송 대기 중이던 모든 임신 관련 알림·리마인더를 시스템 레벨에서 완전히 취소한다. 대상: 주차 변경 축하('이번 주 아기는 ○○ 크기예요'), 산전검진 D-day 리마인더(NT·쿼드·정밀초음파·임당검사 등), 태동 기록 권유, 체중 기록 리마인더, 체크리스트 마감, 태교/태담 권유, 주간 하이라이트 등. '하드'의 의미는 (1) 미래 예약 알림 취소뿐 아니라 (2) 전환 시점에 이미 발송 큐에 들어가 있거나 디바이스에 도착 예정인 알림까지 봉쇄하고 (3) 이후 어떤 경로로도 임신 알림이 재예약되지 않도록 재발화 자체를 차단한다는 것. 사용자가 며칠 뒤 '아기가 자몽만큼 컸어요' 푸시를 받는 일이 절대 없어야 한다.
- **어떻게 보이나**: 사용자에게 별도 화면 없음(백그라운드 동작). 단, 조용한 모드 진입 안내 시트의 '바뀌는 것' 요약에 '임신 관련 알림은 모두 멈춰요'를 명시해 안심시킴. 설정의 알림 토글 화면에서도 임신 알림 항목들이 비활성·회색 처리되어 '꺼져 있음'이 보이게.
- **입력**: 사용자 입력 없음(전환 시 자동 일괄 취소). 사용자가 조용한 모드를 해제(임신 재개·복구)하지 않는 한 재예약 없음.
- **한국 맥락**: 한국 사용자는 카카오·문자 등 외부 알림에 민감하고, 잘못 온 임신 축하 푸시는 공개된 장소에서 보일 경우 극심한 2차 상처가 된다. 특히 가족 공유 중이면 파트너 기기에도 동일하게 하드취소가 전파되어야 함(아래 데이터 보존·공유 항목과 연동). '알림이 다시 올까봐 앱을 지운다'를 막는 것이 이 기능의 신뢰 핵심.
- **엣지/주의**: (1) 가족 공유 시 파트너 기기의 로컬 예약 알림은 파트너 앱이 손실 상태를 동기화한 다음에야 취소되므로, 동기화 지연 동안 파트너에게 임신 푸시가 갈 위험 → 손실 상태를 알림 무력화 우선순위로 동기화. (2) 위젯의 일 단위 자동 갱신도 '알림 유사 능동 표시'로 간주해 함께 정지. (3) 앱 미실행 상태에서 서버 발 푸시(주간 하이라이트 등)가 예약돼 있으면 서버 측에서도 손실 사용자 제외 — 클라이언트 취소만으로 부족. (4) 알림 권한이 꺼져 있던 사용자도 동일 처리(권한 상태 무관하게 재예약 0 유지).

### 데이터 보존 기본값 + 명시적·되돌릴 수 있는 삭제만

- **무엇을**: 조용한 모드는 어떤 경우에도 사용자 데이터를 자동 삭제하지 않는다. 손실 전까지 쌓인 모든 기록(태명, 주차 진행 이력, 태동 세션, 체중 추이, 초음파 사진 첨부, 증상 일지, 메모, 검진 기록)은 그대로 보존되어 '이전 임신' 보관함에서 언제든 다시 볼 수 있다. 삭제는 오직 사용자가 직접, 명시적으로, 그리고 충분한 안내(되돌릴 수 없음 경고)를 받은 뒤에만 가능하며, 기본 동선 어디에도 삭제 버튼을 노출하지 않는다(보관함 깊숙한 곳에서만). 또한 손실 직후 감정이 격할 때의 충동 삭제를 막기 위해, 삭제는 '지금 정말 지우시겠어요? 한번 더 생각하셔도 괜찮아요' 식의 2차 확인과 함께 제공하고, 가능하면 일정 기간 보관(소프트 삭제) 후 영구 삭제 구조로 후회를 되돌릴 여지를 둔다.
- **어떻게 보이나**: 설정 > 이전 임신(보관함)에 손실된 임신이 차분한 톤으로 나열(결과별 잎사귀 아이콘·회색조, 진행 숫자 강조 금지). 항목 탭 시 상세(태명·함께한 기간·보관된 기록 요약·첨부 사진). 삭제는 상세 화면 최하단의 옅은 텍스트 버튼 1개로만, 누르면 '되돌릴 수 없음' 명시 + 보관 기간 안내 + 2단계 확인. 손실 직후 진입 시트·홈·보관함 어디에도 '삭제'를 1급으로 노출하지 않음.
- **입력**: 삭제는 사용자가 명시적으로 트리거할 때만. 그 외 보존은 입력 0(기본 동작).
- **한국 맥락**: 프로젝트 전체 불변 규칙 '데이터 삭제 절대 금지(사용자 명시 외)'와 정합. 한국 사용자 피드백 사례('임신모드 데이터가 사라져 상처')를 반영해, 손실 후 데이터 유실은 가장 치명적인 신뢰 파괴다. 추모 맥락에서 태명·초음파 사진은 유족에게 유일한 유형의 기억일 수 있어 보존 우선순위가 절대적.
- **엣지/주의**: (1) 가족 공유 중 소유자가 삭제하면 파트너 화면에서도 사라지므로, 삭제 전 '공유받은 분도 더는 못 보게 됩니다' 고지 필요. (2) 첨부된 초음파 이미지(클라우드 저장)도 삭제 대상에 포함되는지 사용자가 명확히 인지하도록 분리 안내. (3) 손실 직후 충동 삭제 방지를 위한 소프트 삭제·유예 기간이 있다면, 그 기간 동안에도 데이터가 알림·홈에 능동 노출되지 않아야 함(보존≠재노출). (4) 앱 삭제 후 재설치 시 보관함이 복원되는지(서버 보존) 안내 — 로컬만 보존이면 기기 분실=영구 상실 위험.

### 선택적 추모 기록 (강제·자동 프롬프트 절대 금지)

- **무엇을**: 원하는 사용자만, 원하는 만큼, 아이를 기억하는 공간을 직접 만들 수 있는 조용한 기능. 태명, 함께한 날짜/기간, 짧은 편지·메모, 사진(초음파나 직접 올린 이미지)을 자유롭게 남길 수 있다. 핵심 설계 원칙은 '절대 먼저 권하지 않는다'이다 — 손실 전환 직후 '추모 기록을 작성하시겠어요?' 같은 자동 프롬프트·배지·완료율·체크리스트·리마인더를 일절 띄우지 않는다. 추모 공간은 사용자가 보관함에서 스스로 발견해 열 때만 등장하며, 비워둔 채로 두어도 어떤 독촉이나 빈 상태 강조('아직 작성 안 함')도 하지 않는다. 작성한 내용은 전적으로 사적이며(공유 여부도 사용자가 선택), 일정 형식을 강요하지 않는 자유로운 빈 캔버스에 가깝다.
- **어떻게 보이나**: 보관함 상세 화면 안의 조용한 섹션으로만 존재. 비어 있을 때는 '여기에 ○○(태명)와의 기억을 남기실 수 있어요. 원하실 때, 원하시는 만큼이면 됩니다.' 정도의 한 줄 초대 문구만(빈 상태를 결핍으로 묘사 금지). 입력은 자유 텍스트 + 사진 추가 + 날짜(선택). 저장 후에는 차분한 카드 형태로 표시. 자동 팝업·홈 진입 유도·푸시 0. 손실 전환 플로우의 어느 단계에서도 '추모 기록 만들기'를 다음 단계로 강제하지 않음(전환 완료 화면은 위로 문구로 끝내고, 추모는 사용자가 나중에 스스로 찾도록).
- **입력**: 전부 선택 입력: 태명(보존된 값 자동 채움, 수정 가능), 날짜(선택), 편지/메모 자유 텍스트(선택), 사진(선택). 아무것도 입력하지 않아도 완결된 상태로 간주.
- **한국 맥락**: 한국의 태명 문화(임신 초기부터 아이에게 이름을 붙여 부르는 문화)와 태담(아이에게 말을 건네는 문화)을 반영해, 추모 기록을 '아이에게 건네는 마지막 편지' 톤으로 자연스럽게 받을 수 있게 한다. 한국에는 e하늘 디지털 추모서비스 등 공적 추모 인프라가 있으나 임신 초기 손실은 제도적 추모 대상에서 빠지기 쉬워, 앱 내 사적 추모 공간의 가치가 크다. 종교·기일 문화가 다양하므로 특정 의례(제사·기도 등)를 전제하지 않는 중립적 캔버스 유지.
- **엣지/주의**: (1) 자동 프롬프트 금지 원칙 때문에, 추모 기능의 '발견 가능성'이 낮아질 수 있음 — 보관함 진입 시 한 번, 비강압적 안내 한 줄로만 인지시키고 그 이상 푸시 금지(발견성과 비강압 사이 균형). (2) 사진 첨부 시 초음파 이미지가 의료정보로 민감 — 외부 공유 시 경고. (3) 추모 기록을 파트너와 공유할지 여부는 별도 선택(한쪽은 보고 싶고 한쪽은 보고 싶지 않을 수 있음). (4) 추모 기록 작성을 '완료해야 할 일'로 보이게 하는 진행률·배지·완료 체크 일절 금지(이 영역에서 게임화 요소 전면 배제).

### 앱 평가·업셀·게임화 전면 억제 (조용한 모드 동안)

- **무엇을**: 조용한 모드에 진입한 사용자에게는 상실의 시간 동안 어떤 마케팅·전환·성취 자극도 노출하지 않는다. 구체적으로 (1) 앱스토어 리뷰 요청 팝업(AppReviewPromptService 트리거)을 손실 전환 시점부터 영구 또는 충분히 긴 기간 차단하고 — '기록 20개 도달' '리포트 완료' 같은 기존 트리거가 손실 직후 우연히 발화하는 것을 막는다. (2) 배지·축하 스낵바·완료율·연속 기록 streak 등 게임화/성취 알림을 손실 관련 화면과 시점에서 억제한다. (3) (BabyCare는 광고0·커머스0이므로 상업 업셀은 원래 없으나) 어떤 기능 추천·구독 유도·프리미엄 안내도 이 맥락에서는 노출하지 않는다. 핵심은 '이 사람이 지금 평가·성취·구매를 권유받을 상태가 아니다'를 시스템이 인지하고 모든 권유성 표면을 끄는 것.
- **어떻게 보이나**: 사용자에게 보이는 UI 없음(억제는 비노출로 구현). 손실 전환·추모·보관함 등 손실 경로 화면에서는 별점 요청·배지 스낵바·축하 애니메이션·CTA 배너가 단 하나도 뜨지 않음. 일반 모드로 명시 복귀(임신 재개/새 임신 등록)하기 전까지 권유성 표면 0 유지.
- **입력**: 사용자 입력 없음. 조용한 모드 상태 자체가 억제 플래그로 작용. 해제는 사용자가 명시적으로 일반 흐름에 복귀할 때만.
- **한국 맥락**: 한국 앱 시장에서 별점 요청 팝업은 매우 흔하고 기계적으로 발화되는데, 유산·사산 직후 '앱이 마음에 드시나요? ★★★★★' 가 뜨는 것은 회복 불가능한 신뢰 손상이자 SNS에 박제될 평판 리스크다. '신규 임산부 정복' 목표일수록, 손실을 가장 존엄하게 다루는 앱이라는 평판이 곧 차별화이자 신뢰 자산이 된다.
- **엣지/주의**: (1) 기존 리뷰 트리거가 손실과 무관한 baby 데이터(공존 .both)에서 발화될 수 있음 — 손실 전환 직후 일정 기간은 전체 리뷰 요청을 보류하는 안전 마진 필요(임신 손실과 육아 활동이 한 사용자 안에 공존). (2) 억제가 '한 번 손실하면 영영 리뷰 요청 0'이 되면 정상 회복 사용자에게도 영구 침묵 — 명시적 복귀(새 임신/육아 정착) 시 합리적으로 재개하되, 재개 트리거가 손실 기억을 건드리지 않도록. (3) 배지 시스템이 save path에 깊이 엮여 있어, 손실 경로 저장 동작(추모 기록 저장 등)이 실수로 배지를 발화시키지 않도록 손실 컨텍스트에서 평가 자체를 건너뛰어야 함.

### 정서적 톤·카피 가이드 + 회복 자원 안내 (의료감수 반영)

- **무엇을**: 손실 경로 전반에 적용되는 일관된 언어·톤의 단일 기준과, 사용자가 원할 때 조용히 닿을 수 있는 신뢰 가능한 회복 자원 안내. 톤 원칙: (1) 짧고 담담하게, 과장된 위로·감탄사·이모지 금지. (2) 귀책·실패·원인 추궁 단어 금지('실패' '잘못' '왜' '그래도' '벌써' 등). (3) 사용자를 재촉하거나 다음 단계로 등 떠밀지 않음('천천히 하셔도 괜찮아요'). (4) 결과를 단정하거나 미래를 약속하지 않음('다음엔 잘될 거예요' 금지). 자원 안내는 강요 없이 한 곳에 모아, 원하는 사람만 펼쳐 보도록 한다: 한국 산모의 신체 회복 관련 일반 정보(의료감수 텍스트, 진료 권유는 '의료진과 상담하세요' 수준), 유산·사산 후 받을 수 있는 제도(보건소 산모·신생아 건강관리 지원 — 임신 16주 이상 유산·사산 시 신청 가능, 근로기준법상 유산·사산휴가), 정서적 도움을 받을 수 있는 외부 전문 상담 연결 안내. 모든 의료성 문구에는 면책 문구를 둔다.
- **어떻게 보이나**: 톤 가이드는 화면이 아니라 손실 경로 전 카피에 일관 적용되는 내부 기준(진입 시트·freeze 안내·보관함·추모·완료 화면 문구 전부). 회복 자원은 보관함 또는 조용한 모드 안의 접힌 섹션('도움이 필요하시면' — 기본 접힘, 사용자가 펼칠 때만)으로 제공: 신체 회복 일반 안내(의료감수+면책), 신청 가능한 제도 요약(보건소·휴가) + 외부 링크/연락처, 전문 상담 안내. 자동 푸시·배너로 들이밀지 않음.
- **입력**: 사용자 입력 없음(읽기 전용 안내). 자원 섹션은 사용자가 펼칠 때만 표시.
- **한국 맥락**: 한국 구체 사실로 정확히: 보건소 산모·신생아 건강관리 지원은 임신 16주 이상 유산·사산도 대상이며 의사 소견서/사산증명서 첨부, 확인일 기준 신청 기한 존재(상세는 관할 보건소 안내로 연결). 근로기준법 유산·사산휴가는 임신주수별 차등(15주 이내 10일 / 16~21주 30일 / 22~27주 60일 / 28주 이상 90일), 최초 60일(다태아 75일) 유급, 배우자 출산휴가 별도. 표준 모자보건수첩(산모수첩)에도 관련 안내가 있음을 언급 가능. 단, 제도 수치는 변동되므로 앱은 '대표 기준 + 정확한 내용은 보건소/고용센터 확인'으로 안내하고 단정하지 않음.
- **엣지/주의**: (1) 의료감수 콘텐츠라도 개인 상태 진단·처방 톤 금지(면책 필수, '의료진과 상담' 수준). (2) 제도 수치(휴가 일수·지원 금액)는 법령 개정으로 바뀌므로 하드코딩된 단정 문구는 stale 리스크 — '대표적으로 ○○, 정확한 내용은 보건소/고용센터 확인' 패턴. (3) 외부 상담 연결은 한국에 임신 손실 특화 공적 상담 채널이 분산되어 있어, 신뢰 가능한 출처(보건소·정신건강복지센터 등)만 연결하고 검증되지 않은 민간 채널 노출 금지. (4) 임신중지(중지) 케이스는 사회적 낙인이 강해 카피가 결과 유형을 들추지 않도록 더욱 주의 — 같은 조용한 톤을 유지하되 '유산'을 전제하는 위로 문구가 중지 사용자에게 오노출되지 않도록 유형별 카피 분기는 내부에서만.

## 영역 J — 알림 (절제 원칙 기반 임신 추적기 알림 시스템)

### 알림 철학 — '주 1회 + 임박 1건' 기본값 (절제 원칙)

- **무엇을**: 경쟁사 최대 불만이 '알림 과다'이므로, BabyCare 임신 알림의 출발점은 '거의 안 울린다'이다. 기본 활성 알림은 단 두 갈래뿐: ①주차 전환(주 1회) ②다가오는 산전검진 1건(있을 때만, 1회). 나머지(태동 리마인더, 체크리스트, 체중, 디데이 카운트다운 등)는 전부 기본 OFF이거나 사용자가 직접 켜야 켜진다. 알림은 '엄마가 놓치면 의료적/행정적 손해가 나는 것'에만 쓰고, '들여다보면 좋은 콘텐츠'는 알림이 아니라 홈 카드/배지로 전달한다. 하루 최대 1건 상한(검진 임박+주차 전환이 같은 날 겹치면 하나로 묶음), 야간(21:00~08:00) 발송 금지, 동일 카테고리 연속 무반응 시 자동 빈도 하향(스누즈 학습) 같은 글로벌 가드를 시스템 전체에 강제한다.
- **어떻게 보이나**: 설정 > 알림 상단에 '베이비케어는 알림을 최소한으로 보냅니다. 정말 중요한 것만 알려드려요'라는 한 줄 약속 카피 + 현재 켜진 알림 개수 요약('현재 2종 켜짐'). 첫 임신 등록 직후 단 한 번 '알림 받기' 권한 요청을 띄우되, 거부해도 앱 기능은 100% 동작(검진/주차는 홈 카드로 항상 확인 가능)함을 명시. 권한 거부 후에는 다시 시스템 권한 팝업을 띄우지 않고 설정 화면 안내 배너로만 유도.
- **입력**: 임신 등록 시점, 사용자의 알림 권한 응답, 각 알림 카테고리 토글 상태(설정에 저장), 사용자의 과거 알림 무반응 이력(스누즈 학습용).
- **한국 맥락**: 한국 육아/임신앱 사용자는 알림 스팸에 매우 민감하고 '광고성/유도성 푸시'에 즉시 알림 전체를 꺼버린다. 광고0·커머스0 원칙과 맞물려, 알림에는 어떤 상업 문구·추천 상품·이벤트도 절대 넣지 않는다. 알림 본문은 항상 '엄마와 아기의 상태' 또는 '엄마가 해야 할 행정/의료 액션'만 담는다.
- **엣지/주의**: 권한 거부 사용자: 모든 알림 기능은 무음으로 비활성, 단 홈 카드는 동일 정보를 표시해 기능 손실 0. 알림 권한을 나중에 시스템 설정에서 켠 경우: 앱이 포그라운드 복귀 시 1회 감지해 '알림을 켜셨네요, 주차 알림을 받으시겠어요?' 부드러운 안내. 출산/유산/사산/임신중절로 임신 종료 시: 모든 임신 알림 즉시·완전 중단(특히 검진/태동 리마인더가 종료 후에도 울리면 큰 정서적 상해 — 종료 트랜지션과 동시에 모든 예약 알림 취소). 다태아(쌍둥이 이상)·앱 미사용 장기 휴면 사용자도 동일 상한 적용.

### 주차 전환 알림 (Weekly milestone — 주 1회, 기본 ON)

- **무엇을**: 임신 주차가 NN주0일로 새로 넘어가는 날(주차 시작일) 아침에 단 1건, '이번 주 아기 소식'을 전한다. 본문은 보유 자산 pregnancy-weeks.json의 해당 주차 fruitSize(한국 태아 크기 비유)+milestone(발달 이정표)을 그대로 활용: 예) '14주에 들어섰어요. 아기가 레몬만 해졌고, 표정을 짓기 시작했대요.' 탭하면 홈의 임신 카드(또는 주차 상세)로 진입. 이 알림이 임신 추적기의 '심장박동' — 매주 정확히 1회, 같은 요일/시각에 와서 예측 가능하고 신뢰를 만든다. 임신 전체 기간 중 사용자가 가장 기다리게 되는 긍정적 알림이므로 기본 ON으로 두되, 빈도 조절(매주/격주/끄기)을 제공한다.
- **어떻게 보이나**: 푸시 1줄 본문 + 태아 크기 이모지/일러스트 썸네일. 제목 'NN주N일 · D-XXX', 본문은 크기비유+이정표 1문장. 절대 여러 정보를 욱여넣지 않음(1알림=1메시지). 설정에서 '주차 알림: 매주 / 격주 / 끄기' 세그먼트 + '받는 시각'(기본 오전 9시, 변경 가능) 제공. 탭 → 홈 임신 카드 디프 링크.
- **입력**: LMP/EDD로 산출한 주차 경계(매주 같은 요일이 주차 전환일), pregnancy-weeks.json의 주차별 fruitSize/milestone, 사용자가 설정한 수신 시각·빈도.
- **한국 맥락**: 한국식 'NN주N일 + D-day' 표기를 제목에 항상 병기(한국 임산부는 주차와 디데이를 함께 본다). 태아 크기 비유는 한국 사용자에게 친숙한 과일/사물(자산의 양귀비씨~호박/멜론)로 유지. 4주 미만/40주 초과 구간은 자산에 데이터가 없으므로 알림 대신 일반 격려 문구로 폴백.
- **엣지/주의**: 임신 등록이 늦어 이미 중기/후기인 경우: 등록 즉시 현재 주차 1건만 보내고 다음 주부터 정상 주기. 출산예정일 경과(40주+) 시: '예정일이 지났어요. 매일 태동을 살피고 병원과 연락을 유지하세요' 같은 안전 격려로 전환, 크기비유 중단. 격주 설정 시 짝수 주만 발송. 같은 날 검진 임박 알림과 겹치면 묶어서 1건으로(주차 소식 + '내일 정기검진' 한 카드).

### 산전검진 임박 알림 (한국 표준 검진 일정 기반 — 기본 ON, 1건 1회)

- **무엇을**: 한국 산전검진 표준 일정을 시스템이 알고 있다가, 해당 검진 '권장 주차 창'에 진입하기 직전 또는 사용자가 등록한 다음 검진 예약일 전날(또는 당일 아침)에 단 1건 리마인드한다. 두 소스: ①사용자가 캘린더/건강 탭에 직접 등록한 '다음 산전 예약'(가장 우선) ②미등록 시, 표준 일정에 따른 '슬슬 ○○검사 받을 시기예요' 안내. 표준 마일스톤: 초기 첫 진찰(6~7주), NT(목덜미투명대, 11~13주), 쿼드검사(15~20주), 중기 정밀초음파(20~28주, 통상 20주경), 임신성당뇨 선별(24~28주), 후기 정기검진 간격 변화(28주부터 2주마다·36주부터 매주) 안내. 이 알림은 '놓치면 의료적 기회를 잃는' 정보라서 알림으로 보낼 가치가 있는 두 번째 축이다.
- **어떻게 보이나**: 예약 등록 건: 전날 또는 당일 아침 '내일/오늘 OO병원 정기검진이에요. 산모수첩·국민행복카드 챙기세요'. 표준 일정 건: 검진 창 진입 시 1회 '11~13주는 1차 기형아검사(NT) 시기예요. 아직 예약 전이면 병원과 일정을 잡아보세요'. 각 검진 알림 하단/연결 상세에 '이 검사는 무엇을 보나요'를 의료감수 문구로 1~2줄(예: NT=다운증후군 등 염색체이상 선별). 탭 → 캘린더의 해당 예약 또는 검진 가이드 상세.
- **입력**: 사용자 등록 예약일(있으면 우선), LMP/EDD 기반 현재 주차, 한국 표준 검진 주차 창 테이블, 사용자가 이미 '완료'로 체크한 검진(중복 알림 방지).
- **한국 맥락**: 검진 항목·주차 창은 검증된 한국 표준 사용: NT 11~13주, 쿼드 15~20주, 정밀초음파 20~28주, 임당 24~28주, 정기검진 빈도(28주까지 4주/36주까지 2주/이후 매주). 검진 알림에 '산모수첩'과 '국민행복카드(바우처)' 지참 안내를 자연스럽게 결합(한국 병원 실무 동선). 의료 문구는 산부인과 감수자 검수 콘텐츠만 사용하고 진단·판단 표현 금지(면책 문구 필수).
- **엣지/주의**: 이미 해당 검진을 완료 체크한 경우: 표준 일정 알림 스킵. 검진 창을 이미 지나친 늦은 등록: '지난 검사' 폭격 금지 — 현재 주차 이후의 다가올 검진만 안내. 예약 등록 건과 표준 안내가 같은 검진을 가리키면 예약 건만 발송(중복 제거). 후기 매주 검진 구간(36주+)에서도 매주 알림 폭격하지 않고, '검진 간격이 매주로 짧아졌어요' 1회 안내 후엔 사용자 등록 예약 기반으로만 리마인드.

### 태동 모니터링 리마인더 (28주+ 옵트인, 기본 OFF)

- **무엇을**: 태동 카운트(킥 세션) 자산과 연동해, 후기 임신에서 '오늘 태동을 살펴볼까요' 부드러운 리마인더를 보낸다. 단 의료적 민감도가 높고(불안 유발 가능) 기본 OFF — 사용자가 명시적으로 켜야 동작. 켜면 하루 1회(사용자 지정 시각, 기본 저녁), '태동이 활발한 시간에 10분만 아기 움직임을 느껴보세요'. 태동 카운트 기능을 쓰는 사용자에게만 가치가 있으므로, 킥 세션을 1회라도 기록한 사용자에게만 설정에 노출(미사용자에게는 토글 자체를 숨겨 알림 메뉴를 가볍게 유지).
- **어떻게 보이나**: 설정 > 알림에 '태동 살피기 알림' 토글(28주 이상 + 킥 세션 사용 이력 있을 때만 노출) + 받는 시각 선택. 본문은 강박 유발하지 않는 격려 톤('의무'가 아닌 '함께 느껴보기'). 탭 → 건강 탭 태동 카운트(킥 세션) 화면. 카운트 결과가 평소보다 현저히 적을 때의 대응은 알림이 아니라 화면 내 의료감수 안내로 처리(예: '30주 이후 태동이 평소보다 뜸하면 병원과 상의하세요' — 자산 disclaimer.kick 활용).
- **입력**: 현재 주차(28주 게이트), 킥 세션 기록 존재 여부(노출 조건), 사용자 지정 수신 시각, 토글 상태.
- **한국 맥락**: 한국·국제 표준상 태동 인지는 16~18주(경산모 더 이름), 본격 모니터링 권고는 30주 전후부터(시간당 평균 3회 이하·하루 20회 미만이 2일 지속 시 내원). 게이트를 28주로 두어 너무 이른 불안 조장 방지. 알림은 '횟수 강제'가 아니라 '습관 제안'으로 — 한국 임산부 사이 NST(비수축검사, 32주+) 정서와도 충돌하지 않게 부드럽게.
- **엣지/주의**: 28주 미만: 토글 숨김(설정에 아예 안 보임). 킥 세션 미사용자: 토글 숨김. 켠 뒤 출산/임신 종료: 즉시 영구 취소. 사용자가 알림을 받고도 3회 연속 카운트를 안 하면 '이 알림이 부담되면 끌 수 있어요' 1회 안내 후 자동 빈도 하향 또는 OFF 제안(스누즈 학습). 불안장애 고려해 '태동 적음=이상' 식의 단정적 알림 본문 절대 금지.

### 행정·준비 마일스톤 알림 (시기성 강한 1회성, 선택적)

- **무엇을**: 임신 중 '시기를 놓치면 손해'인 한국 특유의 행정/준비 액션을, 해당 시기에 딱 1회씩만 알린다. 항목 예: ①임신 확인 직후 '국민행복카드(임신·출산 진료비 바우처) 신청하기' ②안정기 진입(12~13주) 무렵 '직장 임신 사실 알림·태아검진휴가 확인' 류 일반 안내 ③후기(32~36주) '출산가방 준비' '출생신고/산후조리 계획' ④만삭(36주+) '만삭사진·태교 마무리'. 이들은 전부 체크리스트 자산(targetWeek)과 1:1로 연결되어, 체크리스트 항목이 완료되면 해당 알림은 발송하지 않는다. 빈도가 낮고(전 기간 합쳐 몇 건) 강한 시기성을 가져 절제 원칙과 양립.
- **어떻게 보이나**: 기본은 '준비 알림 받기' 단일 토글(ON/OFF). 켜면 각 마일스톤 창 진입 시 1회. 본문은 행동 1개만('국민행복카드를 아직 신청 안 하셨다면 지금 신청해보세요. 진료비 바우처로 산전검진 비용을 지원받아요'). 탭 → 체크리스트의 해당 항목. 상업적 추천(특정 카드사·산후조리원 등) 절대 금지 — 제도/행동만 안내.
- **입력**: 현재 주차, 체크리스트 항목의 targetWeek와 완료 여부(완료 시 해당 알림 억제), '준비 알림' 토글 상태.
- **한국 맥락**: 국민행복카드는 임신확인서 발급 직후 신청 가능(요양기관 정보입력→카드사 바우처 신청)이라 '초기 1회' 안내가 실제로 유용. 출산가방·만삭사진은 한국 임신 문화의 표준 준비물/이벤트라 후기 안내로 적절. 태담(태교 대화) 문화는 강박이 되지 않게 '알림'보다 홈 콘텐츠로 두고, 행정 마일스톤만 알림화.
- **엣지/주의**: 늦은 등록으로 초기 마일스톤(국민행복카드 시기)을 이미 지난 경우: 등록 직후 현재 시점 기준 유효한 것만 1회 묶음 안내(과거 항목 폭격 금지). 체크리스트에서 사용자가 직접 완료 처리한 항목은 알림 영구 스킵. '준비 알림' 토글 OFF면 전체 침묵(체크리스트는 홈/캘린더에서 능동 확인 가능). 임신 종료 시 후기 준비 알림(출산가방 등) 즉시 취소.

### 알림 제어·투명성 UX (끄기/빈도/시각/카테고리별 미세조정)

- **무엇을**: 모든 알림 카테고리를 한 화면에서 개별 제어한다. 각 카테고리(주차 전환 / 검진 임박 / 태동 살피기 / 준비 알림)마다 ON·OFF + 가능 시 빈도·수신 시각. 추가로 글로벌 컨트롤: ①조용한 시간(야간 자동 무음, 기본 21:00~08:00, 사용자 변경 가능) ②한 번에 모두 끄기('알림 전부 끄기' 단일 스위치 — 기능 손실 없이 홈 카드로 대체됨을 명시) ③'알림은 하루 최대 1건' 정책 표기. 스누즈 학습: 같은 카테고리 알림에 N회 연속 무반응이면 '이 알림 계속 받으시겠어요?' 1회 확인 후 자동 빈도 하향. 사용자가 직접 끈 알림은 다시 권유하지 않음(재유도 금지).
- **어떻게 보이나**: 설정 > 알림 = 카테고리 카드 리스트(각 카드: 제목 + 한 줄 설명 + 토글 + 펼치면 빈도/시각). 상단에 '최소 알림 약속' 카피 + '현재 N종 켜짐' 요약. 하단에 '조용한 시간' 시간대 피커 + '모든 알림 끄기'. 권한 미허용 시 'iOS 설정에서 알림을 허용해야 받을 수 있어요' 배너(딥링크). 모든 토글 변경은 즉시 반영(다음 예약부터 적용).
- **입력**: 각 카테고리 토글·빈도·시각 설정값, 글로벌 조용한 시간, 마스터 OFF 스위치, 무반응 카운터(스누즈 학습), iOS 알림 권한 상태.
- **한국 맥락**: 한국 사용자는 '한 번에 다 끄기'와 '시간대 설정'을 강하게 선호(야간 푸시에 특히 민감). '왜 이 알림을 보냈는지' 명확한 라벨링이 신뢰를 만든다. 어떤 토글도 기본값이 '몰래 ON'이 되지 않도록 — 옵트인 항목은 명시적으로 OFF에서 시작.
- **엣지/주의**: 마스터 OFF 중 임신 종료: 추가 동작 없음(이미 무음). 조용한 시간에 걸린 알림: 다음 허용 시각(08:00)으로 지연 발송하되, 시기가 지나면(예: 어제 검진) 폐기. 사용자가 수신 시각을 야간으로 직접 지정하면 조용한 시간 경고 1회 후 사용자 선택 존중. 알림 전부 끈 사용자에게는 '알림 켜세요' 푸시를 절대 보내지 않음(앱 내 배너로만, 그것도 비강압적으로).


## K. 정복 추가 기능 (커뮤니티 · 태교 콘텐츠 · 운동) — PO 2026-06-11 추가

### 커뮤니티 (출산예정월 동기 그룹)

- **무엇을**: 같은 출산예정월(예: "2026년 12월 출산맘") 산모들이 모여 글·댓글·질문을 나누는 커뮤니티. 주차가 비슷한 또래와 불안·정보·후기를 공유. KR 리더(베이비빌리 200만·마미톡)의 핵심 retention 엔진.
- **어떻게 보이나**: 별도 진입(탭 또는 홈 커뮤니티 카드). 예정월 그룹 자동 배정 + 주제별 전체 게시판(검진후기·증상·출산준비 등). 글 목록 → 상세 → 댓글. 익명 닉네임.
- **입력**: 글 작성(텍스트+사진), 댓글, 공감, 신고/차단.
- **한국 맥락**: 네이버 맘카페 status quo를 앱 내로. "○월맘" 문화 = 예정월 그룹. **광고0이 차별점**(맘카페·경쟁앱 광고 피로 회피).
- **엣지/주의**: ⚠️ **운영·모더레이션 부담 최대** — 신고/차단/금칙어/스팸·상업글 차단 필수. 의료 오정보 확산 위험(면책+신고 동선). **손실(유산/사산) 사용자는 조용한 모드 시 커뮤니티 노출 억제.** cold-start(초기 빈 그룹) → 통합 그룹/시드 전략. 임신 사실 노출 프라이버시 주의. 운영 인력/정책 선결.

### 태교 음악/동화

- **무엇을**: 태교용 음악(클래식·자장가·자연음)·동화·태담 가이드 재생. 매일 듣고 아기에게 들려주는 루틴.
- **어떻게 보이나**: 건강 탭/정서·기록 영역 '태교' 섹션 + 플레이어(재생목록·백그라운드·잠금화면 컨트롤·취침 타이머). 주차별 추천 태교 콘텐츠.
- **입력**: 재생/일시정지/목록, 즐겨찾기, 타이머. (본인 음원 추가는 저작권상 범위 밖)
- **한국 맥락**: 한국 태교 문화 핵심(클래식 태교·동화 읽어주기). 베이비빌리·마미톡 보유. **아빠가 동화 읽어주는 태담(공유)과 연결.**
- **엣지/주의**: ⚠️ **음원·동화 저작권/라이선스 확보 필수**(커머스 아님, 라이선스 비용 발생). 자체제작 vs 라이선스 결정. 손실 후 자동 추천/재생 억제.

### 운동/요가 가이드 〔의료감수〕

- **무엇을**: 임신부 안전 운동·요가·스트레칭·호흡/이완 가이드(영상 또는 이미지+설명). 삼분기·주차별 권장/금기 운동.
- **어떻게 보이나**: 건강 탭 '운동' 섹션. 운동 목록(부위·주차·난이도) → 상세(동작·주의). 주차별 추천.
- **입력**: 운동 보기/완료 체크(선택)·즐겨찾기. (추적 A의 활동과 연계 가능)
- **한국 맥락**: 임산부 요가·필라테스·짐볼 인기. 금기 동작(후기 반듯이 눕기·복압 등) 의료감수 명시.
- **엣지/주의**: ⚠️ **의료 책임** — "운동 전 의료진 상담" 면책 필수. 고위험 임신(조기진통·전치태반 등) 운동 금기 안내. 부상 위험 동작 경고. 영상 제작/라이선스 비용.
