## 운영 원칙 (이 이니셔티브 표준)

- **항목당 독립 브랜치 + PR** (순서 무관·병렬 가능하나 실행은 순차 — 시뮬레이터 경합·리뷰 가능성 유지)
- **TDD**: 순수 로직 RED→GREEN → 배선 → `make verify` ALL PASS 후 커밋. 커밋 전 branch 실게이트(`[ ... ]`).
- **파이프라인(push→PR→CI→머지)은 PO의 "전부 다 고치라" 지시로 표준 승인 간주.** 예외(개별 확인 필수): D2 프로드 쓰기, C6 RC 활성화, 의료 카피 신설 시 감수 플래그.
- 카피는 의료 압박 금지(safety.md). 신규 컬렉션 생기면 Narrow Protocol 5단계. `print()` 금지.
- 죽은 코드 제거 전 grep 전수 재확인(gap-analyzer 재검증 원칙).
- **TF 업로드는 이니셔티브에 포함 안 됨** — 빌드 99는 PO 결정 시 [관문] 태스크로.
